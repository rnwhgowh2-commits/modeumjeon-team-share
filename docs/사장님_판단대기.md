### 1. 계수 적용 — 「소싱처 × 브랜드」를 담을 자리가 없습니다

**버튼은 만들었습니다** (사장님 4번 = 나). 누르면 **무엇이 바뀌는지 먼저 보여주고**,
확인해야 저장됩니다. 다만 만들다 보니 틈이 하나 나왔습니다.

**제안은 `(소싱처 × 브랜드)` 단위인데, 계수 규칙 스코프는 넷뿐입니다.**

```
url · model · brand · source     ← 「소싱처 × 브랜드」 조합이 없음

brand 로 걸면   → 그 브랜드가 다른 소싱처에도 걸림
source 로 걸면  → 그 소싱처의 다른 브랜드까지 걸림
```

**지금 처리** — `brand` 로 걸되, 그 브랜드가 **여러 소싱처에 걸쳐 있으면 경고**를 띄웁니다.
「나이키는 SSG 에도 있어 같이 바뀝니다」처럼요. 막지는 않습니다 — 알고 누르시면 됩니다.

**정할 것** — 이대로 둘까요, 스코프를 새로 만들까요?

| | |
|---|---|
| **가** | **지금대로 (brand + 경고)** ← 추천 |
| **나** | `소싱처×브랜드` 스코프 신설 |

> **가를 추천합니다.** 나는 `resolve_crawl_weight` 의 **상속 순서를 건드려야** 하고,
> 그건 라이브 크롤 스케줄러를 또 고치는 일입니다. 경고로 충분히 막힙니다.
>
> 참고 — 지금은 관측이 10~27회라 **등급이 안 나와서 누를 게 없습니다.** 급하지 않습니다.

**사장님이 「데이터코드지도 참고해서 찾아봐」라고 하신 게 맞았습니다.**
앞서 저는 API **본문 문구**만 훑어서 주문 조회 한도 3건만 봤는데,
지도의 `markets` 블록에 **마켓 단위 규칙**이 따로 적혀 있었습니다.

| 마켓 | 지도에 적힌 것 | 성격 |
|---|---|---|
| **쿠팡** | **5 req/s 토큰버킷, 429 Retry-After 존중** | **속도 한도** |
| 스마트스토어 | 계정 순차 조회 (병렬 시 429 위험) | **동시 1개** |
| 11번가 | 계정 순차 조회 필수 — 병렬 시 **429 전체 다운** | **동시 1개** |
| 롯데온 | IP 등록 필수 | 인증 (속도 아님) |
| 옥션 · G마켓 | (비어 있음) | 모름 |

**「초당 몇 개」와 「동시에 몇 개」는 다른 제약입니다.** 속도를 아무리 낮춰도 동시에 두 개를
쏘면 11번가는 전체가 죽습니다. 그래서 **문구를 그대로** 화면에 띄우고,
「한 번에 하나씩」 경고를 따로 붙였습니다(이건 반영 완료).

**정할 것** — 제가 시드한 쿠팡 한도를 고칠까요?

```
지금 (제가 넣은 값)   60초에 50개  = 초당 0.83개   ← 「로켓그로스 주문조회」 한도
지도의 게이트웨이 한도  5 req/s      = 초당 5개      ← 쿠팡 API 전체
```

**제가 좁은 값을 넣었습니다.** 「분당 50회」는 **그 주문조회 API 하나**의 한도인데,
업로드는 다른 API 라 게이트웨이 한도(5 req/s)가 맞습니다.

| | |
|---|---|
| **가** | **5 req/s 로 고침** ← 추천 |
| **나** | 지금(0.83개/초) 유지 |

> **가를 추천합니다.** 지금은 실제 한도의 **1/6 로 묶어둔** 셈이라 업로드가 6배 느립니다.

### 7. H몰·롯데아이몰 「보류」를 풀까요

**코드 문제가 아니었습니다.** 오늘 끝까지 따라가 확인했습니다.

사장님이 「H몰·롯데아이몰은 표면가와 최종매입가 구분이 안 보인다」고 하셨는데,
**화면도 엔진도 멀쩡합니다.**

| 확인한 것 | 결과 |
|---|---|
| 매트릭스 2단 표기 (표면 취소선 / 최종 굵게) | ✅ **이미 있음** (2026-06-24 반영) |
| 문자열 키 소싱처(`key:hmall`) 혜택 로딩 | ✅ **이미 고침** (M1-6) |
| **H몰·롯데아이몰 혜택 데이터** | ❌ **일부러 안 넣음** |

혜택이 0이면 최종매입가 = 표면가라 **2단이 뜰 이유가 없습니다.** 화면은 「같으면 한 줄」로
일부러 만들어져 있습니다(중복·모순 방지).

**왜 안 넣었나** — 코드에 사장님 지시가 그대로 적혀 있습니다.

```
H몰(현대H몰) 2.7% / 롯데홈쇼핑 2.5%(=롯데아이몰) — 사장님 보류 지시
카드 청구할인: 라이브 확인된 건은 롯데아이몰 삼성 7% 하나뿐인데
              그 소싱처가 보류 대상 → 시드 비워둠
```

**정할 것** — 이 보류를 풀까요?

> **추천 — 푸시는 게 맞습니다.** 지금은 두 소싱처의 매입가가 **실제보다 비싸게** 잡혀 있습니다.
> 매입가가 비싸게 잡히면 마진이 작게 보여 **팔 수 있는 걸 안 파는** 손해가 납니다.
> (반대로 싸게 잡히면 역마진이라 더 위험하니, 지금 방향 자체는 안전한 쪽입니다.)
>
> 풀기로 하시면 넣을 값 — H몰 캐시백 2.7% · 롯데아이몰 캐시백 2.5% ·
> 롯데아이몰 삼성카드 청구할인 7%. **맞는지 확인만 해주시면** 바로 넣겠습니다.
