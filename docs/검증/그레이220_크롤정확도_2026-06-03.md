## 3. 문제 판정

### 🔴 P1. 롯데온 전면 크롤 실패 (#2, #7) — 데이터 누락
- 원인: AWS 서버에 Playwright 크롬 미설치.
  `BrowserType.launch: Executable doesn't exist at /home/app/.cache/ms-playwright/chromium_headless_shell-1223/...`
- 실제 롯데온은 정상 + 사이즈별 정확수량("2개 남음")까지 제공하는데 전부 누락.
- 영향: 롯데온 가격/재고 0건 반영. 최저가 비교에서 롯데온 통째 빠짐.

### 🔴 P2. SSF 가격 불일치 (#6) — 매트릭스 가격 산출 버그 (금전 영향)
- SSF 어댑터는 **옵션단위로 119,900원(=사이트 정확값)**을 잘 긁었다.
- 그러나 매트릭스가 표시에 **상품단위 `last_price`(122,376)**를 쓴다.
- 즉 **재고는 옵션단위 매칭(`_match_option_stock`)인데 가격은 상품단위(`sp.last_price`)** — 비대칭.
- 영향: SSF 표시가가 실제보다 2,476원 높게 표시 → 마진계산·최저가 판정 오염.

### 🟠 P3. SSG 모음전 URL 크롤 실패 (#5)
- "옵션 추출 실패 — itemId=1000616111568" (`dealItemView.ssg` 딜페이지 구조가 일반 `itemView`와 다름).
- 단, 색상별 `SSG_그레이`(#9 `itemView`)는 ok → 컬럼은 #9로 채워짐(15개). 모음전 URL만 무효.

### 🟡 P4. "재고있음"은 실수량이 아님 (999 센티넬) — 사용자 체감 불일치의 핵심
- 르무통·무신사·SSF가 재고 있는 사이즈를 전부 **999(=있음 더미)**로 저장.
- 화면 "재고있음" = "크롤됨 + 품절 아님"일 뿐, 몇 개 남았는지가 아님.
- 사이트는 "2개 남음/품절임박/품절"을 주는데 크롤러가 안 긁음. (품절=0은 잘 잡음)

### ⚪ P5. 미검증 2건
- 스마트스토어 "1개", SSG "15개"는 사이트 차단으로 실제 대조 못함 → 신뢰도 보류.

## 4. 가격/재고 → UI 표시 로직 (근거 체인)

### 데이터 경로
```
[등록] bundle_source_urls (모델 단위, source_key+url+label)
   └─ option_source_url_links (옵션 ↔ 등록URL N:N)   ← 매트릭스는 이걸로 옵션에 URL 연결
[크롤] 소싱처 어댑터 → SourceProduct(상품단위: last_price/last_stock/last_status)
                      → SourceOption(색+사이즈 단위: current_price/current_stock)
[표시] api_pricing.get_option_matrix → sku_to_sources[sku] = [{...}]
```

### 가격 (현재 동작) — `webapp/routes/api_pricing.py`
- OptionSourceUrlLink 경로(L446~467): `crawled_price = sp.last_price` ← **상품단위 고정** (버그 원천)
- 프론트 `_matrix_v3.html`:
  - `_pickBestSrc(list)`(L4903): 컬럼별 대표 URL 선정 = error아님+품절아님+가격>0 중 최저, 없으면 가격있는것, 없으면 첫째
  - 셀 가격(L4938): `finalPrice = breakdown.final_price ?? src.crawled_price` → 마진/카드 가공 후 표시
  - 즉 표시가의 **기반값이 crawled_price(=상품 last_price)** 라, 옵션가와 다르면 그대로 오염

### 재고 (현재 동작) — 단일 진실 원천 `_resolve_stock`
- OptionSourceUrlLink 경로(L440~445): `_match_option_stock(색+사이즈)` → 실패시 `sp.last_stock` fallback
- `_resolve_stock(site, raw)`(L127): `0→품절`, `None/≥900→재고있음`, `무신사≥10→재고있음`, `그외→N개`
- 프론트(L4953~): 백엔드 `stock_label/stock_qty/stock_out`만 사용(프론트 계산 폐기)
- ✅ 재고는 옵션단위 매칭 → 비교적 정확. 단 999 센티넬 한계(P4).

### 핵심 비대칭 (P2의 근본)
> 재고 = 옵션단위(`_match_option_stock`) · 가격 = 상품단위(`sp.last_price`)
> → 둘을 모두 옵션단위로 맞추면 SSF 불일치 해소.
