## Phase 1 · 가격 계산기 통합 ①

- **목표**: 화면 표시 가격 = 마켓 업로드 가격. 계산을 한 곳에서만.
- **현황 (조사됨)**: 가격 계산이 2곳 — 경로1 `lemouton/pricing/engine.py`(스케줄러용) / 경로2 `webapp/routes/api_pricing.py` `calc_auto_price`(매트릭스 화면용). 마진식·가드레일·적립차감 처리가 달라 값이 어긋남.
- **결정**: 경로2(업로드 기준) 방식으로 통일. 원가 = 혜택 모두 반영된 매입가.
- **작업**
  - 단일 가격 계산 함수 1개로 통합 (혜택 차감 → 원가 → 마진 → 수수료 → 배송비 → 라운딩 → 가드레일)
  - 매트릭스 화면·스케줄러·업로더가 모두 이 함수 호출
  - 마진 3계층(옵션>모델>템플릿)을 양쪽에 동일 적용
- **검증**: 동일 옵션의 화면값과 업로드 payload값이 일치 (실시연)
- **진행 결과 (20260521)**: ✅ 통합 함수 `lemouton/pricing/unified.py` 신설 (pytest 14/14) + 경로②(옵션 매트릭스 `calc_auto_price`) 위임 완료 (회귀 49 passed). 경로③(재고관리)은 분석 결과 '사입 마진 참고가'(수수료·배송 없음)로 별개 계산 → 통합 제외. 경로①(스케줄러 분모형)은 현재 라이브 미사용 → Phase 5(업로드 복구)에서 곱셈형 통합. 남은 것: 브라우저 실시연.

## Phase 2 · 단계형 옵션 · 조합 추가 ②

- **목표**: 옵션을 1~3단계 자유 구성. 단계 이름 자유. 1축 쉼표 / 2축 매트릭스 생성.
- **현황**: `Option` 모델이 `color_code`×`size_code` 2축 고정. `models_v2.py`에 N축 설계(`option1/2/3` + `option_labels`)가 있으나 미사용.
- **작업**
  - 옵션 구조를 N축 가변으로 (단계 설계 저장 + 옵션값 저장) — Alembic 마이그레이션
  - 옵션ID(SKU) 생성 규칙: 단계 값 조합
  - "조합 추가" 모달 확장 — 단계 이름 자유 + 1축 쉼표입력 / 2·3축 매트릭스 체크 생성
  - 옵션 상세에서 개별 추가·수정·삭제
- **검증**: 1/2/3단계 모음전 각각 옵션 생성·수정·삭제 실시연

## Phase 3 · 소싱처 URL 옵션 단위 정리 ③

- **목표**: URL 저장을 옵션 단위로 일원화 + 한 소싱처 다중 URL.
- **현황**: URL 3중 저장 — `Model.url_*` / `BundleSourceUrl` / `OptionSourceUrl`. `OptionSourceUrl`은 `(canonical_sku, source_id)` UniqueConstraint로 1개 제한.
- **작업**
  - `OptionSourceUrl` 단일화 + UniqueConstraint 완화(다중 URL 허용)
  - `Model.url_*` · `BundleSourceUrl` 점진 폐기 (읽기 호환 유지 → 안정 후 제거)
  - 오프라인 전용 옵션 표시 (URL 없음 = 정상)
- **검증**: 옵션 1개에 소싱처 여러 개 + 한 소싱처 URL 여러 개 등록 실시연

## Phase 4 · 마켓 옵션 매핑 ⑤

- **목표**: 우리 옵션 ↔ 마켓 옵션 연결.
- **작업**
  - 매핑 방법 선택 UI — 방법1 API 자동 / 방법2 수동 / 방법3 엑셀
  - 미매칭 처리 — 후보 제안 + 수기 정정 (드롭다운 3단: 추천후보·검색·직접입력)
  - 색상명 사전 학습(선택)
- **현황**: 마켓 어댑터(`shared/platforms/smartstore`·`coupang`)는 실구현. 기존 마켓 탭(신규등록/기존업데이트·API자동/엑셀/수동) 디자인 유지.
- **검증**: API 자동 매칭 → 미매칭 후보·수기·직접입력으로 7/7 연결 실시연

## Phase 5 · 자동 업로드 파이프라인 복구 ④

- **목표**: 자동 업로드 복구. 수동 → 일괄 → 자동 단계적.
- **현황 (조사됨)**: `lemouton/uploader/orchestrator.py` `run_uploader` 시그니처와 호출부 불일치(`api.py`·`scheduler/jobs.py`) → `TypeError`. `scheduler/jobs.py`는 `DryRunAdapter` 고정.
- **작업**
  - `run_uploader` 호출부 시그니처 정합 수정
  - `DryRunAdapter` 고정 해제 → 실 어댑터 배선
  - 1차 옵션별 수동 업로드 → 2차 선택 일괄 → 3차 자동 스케줄
- **검증**: 옵션 1건 수동 업로드 → 마켓 반영 실측

## Phase 6 · 모음전 구성 4탭 재구성 ⑥

- **목표**: `기본 → 템플릿 → 옵션 → 마켓` 4탭.
- **작업**
  - 기존 "소싱처 URL" 탭 제거 (Phase 3에서 옵션 단위 흡수)
  - 옵션 탭 = 조합 추가 + 옵션 통합 관리 (기존 옵션 탭 유지·보완)
  - 가격 산출과정·재고 연결 팝업 등 시안 반영
- **검증**: 4탭 위저드 전 구간 실시연
