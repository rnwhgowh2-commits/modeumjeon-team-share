# 현재 브라우저 기준 가격·혜택 엔진 — 구현 계획 (무신사 르무통 우선)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 무신사 르무통 상품의 표면노출가·혜택·최종매입가를 "현재 열린 브라우저 기준으로만" 수집·계산하고, 현재 크롤 데이터가 없으면 옛 값으로 폴백하지 않고 "크롤실패/혜택 미수집"으로 표면화한다.

**Architecture:** 확장이 현재 페이지 DOM에서 혜택 라인+금액을 긁어(신규) crawl-result에 푸시 → 신선도 마커와 함께 저장 → `compute_breakdown`이 신선도 게이트를 통과한 이번 크롤 스냅샷만으로 계산(폴백 금지). 계산 부품(`gate_benefits`·`compute_final_price`)과 무신사 주입 분기는 이미 존재하므로 "배선 + 신선도 게이트 + 미수집 경로" 추가가 핵심.

**Tech Stack:** Python 3 / Flask / SQLAlchemy / pytest (백엔드), Chrome MV3 확장 JS (크롤러), 기존 매트릭스 JS/HTML (UI).

**설계서:** `docs/superpowers/specs/2026-06-14-현재브라우저기준-가격혜택엔진-design.md`

## 실행 전 필독 — 리스크 / 환경

- **브랜치**: 현재 `feature/crawl-worker-queue`는 origin/main 대비 stale. **이 작업은 origin/main 기준 새 worktree에서** 진행하고, 변경분만 main에 올린다 (메모리 `project_deploy_isolated_change_via_worktree`).
- **확장 버전**: 단일 원본 = `프로그램/_시스템/extension/moum-crawler`. main 계열(v0.4.x) 기준으로만 수정. 버전 문자열 `MOUM_EXT_VERSION` 올릴 것.
- **이중차감 지뢰**: 혜택 모델 변경 시 라이브(같은 Supabase) 구버전 코드와 충돌 주의. 마이그레이션 없는 JSON 키 방식이라 스키마 변경은 없음(아래 저장 방식 참조).
- **저장 방식**: 새 컬럼 추가 ❌ (프로젝트 관례 = JSON 키로 마이그레이션 회피, `crawl_guide.py:16` 주석 근거). 크롤 스냅샷은 `SourceProduct.dynamic_benefits_json` 안의 예약 키 `_crawl`에 저장한다. 기존 동적혜택 소비부는 특정 키(`point_rate` 등)만 읽으므로 `_crawl` 키 추가는 안전.
- **폴백 금지 원칙**: 메모리 `feedback_no_fallback_price_on_match_fail` — 매칭/크롤 실패 시 대표가·템플릿값 폴백 절대 금지.

### 크롤 스냅샷 데이터 형태 (전 Phase 공통 계약)

`SourceProduct.dynamic_benefits_json` JSON 안:

```json
{
  "_crawl": {
    "crawled_at": "2026-06-14T05:00:00+00:00",
    "is_logged_in": true,
    "benefits_ok": true,
    "lines": ["상품 쿠폰 5%", "등급 할인 불가", "무신사 머니 2,000원 적립"],
    "amounts": {
      "상품쿠폰":   {"type": "amount", "value": 6145},
      "등급할인":   {"type": "amount", "value": 0},
      "등급적립":   {"type": "amount", "value": 0},
      "무신사머니 결제 적립": {"type": "amount", "value": 2000}
    }
  },
  "surface_price": 122900
}
```

- `lines` = 게이트(on/off)용 화면 혜택 문구 라인.
- `amounts` = 적용 시 차감/주입할 실제 금액(이번 크롤값). 키 = `crawl_guide.pricing.benefits[].name`과 일치.
- `benefits_ok` = 혜택영역을 실제로 읽었는가(빈 배열을 "혜택없음"으로 둔갑 금지).

```python
def benefits_fresh(snapshot, last_status) -> bool:
    """혜택 크롤 스냅샷을 '이번 브라우저 기준'으로 신뢰할 수 있는가 — 혜택용 게이트.

    is_crawl_valid(표면가)와 동형. 통과 못 하면 혜택은 '미수집'으로 표면화하고
    옛 스냅샷·템플릿 값으로 폴백하지 않는다(데이터 무결성·폴백 금지).
    """
    if not isinstance(snapshot, dict):
        return False
    if not snapshot.get('benefits_ok'):
        return False
    return last_status != 'error'
```

```python
def _build_crawl_snapshot(item: dict, *, now_iso: str) -> dict:
    """확장 크롤 1건(item)에서 '이번 브라우저 기준' 혜택 스냅샷을 만든다.

    benefit_lines/benefit_amounts 가 오면 benefits_ok=True. 없으면 빈 스냅샷
    (benefits_ok=False) — 빈 배열을 '혜택 없음'으로 둔갑시키지 않는다(미수집).
    """
    lines = item.get('benefit_lines')
    amounts = item.get('benefit_amounts')
    has = isinstance(lines, list) and bool(item.get('benefits_ok'))
    return {
        'crawled_at': now_iso,
        'is_logged_in': (None if item.get('is_logged_in') is None
                         else bool(item.get('is_logged_in'))),
        'benefits_ok': bool(has),
        'lines': list(lines) if isinstance(lines, list) else [],
        'amounts': dict(amounts) if isinstance(amounts, dict) else {},
    }
```

```python
    """무신사: 이번 크롤 스냅샷(snap)으로 effective 혜택 리스트를 만든다.

    - snap 없음/None → None (미수집 신호; 호출부가 산출불가 처리).
    - gate_benefits(가이드 키워드, snap.lines, excludes)로 on/off 판정.
    - 금액은 snap.amounts[name] (이번 크롤값). 가이드는 키워드·속성만 제공.
    폴백 금지: 템플릿/옛 값으로 대체하지 않는다.
    """
    if not isinstance(snap, dict):
        return None
    from lemouton.pricing.benefit_gate import gate_benefits
    gated = gate_benefits(guide_benefits or [], snap.get('lines') or [],
                          exclude_keywords or [])
    applied = {g['name'] for g in gated if g['applied']}
    amounts = snap.get('amounts') or {}

    class _Inj:
        def __init__(self, name, btype, value, enabled):
            self.id = -1; self.benefit_name = name; self.benefit_type = btype
            self.value = value; self.enabled = enabled
            self.sort_order = 999; self.template_id = None

        eff.append(('crawl', _Inj(nm, btype, val, enabled=(nm in applied and val > 0))))
```
