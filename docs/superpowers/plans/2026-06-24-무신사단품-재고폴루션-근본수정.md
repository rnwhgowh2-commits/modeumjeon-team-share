# 무신사 단품 재고 폴루션 근본수정 — 구현 플랜

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. 각 스텝 체크박스(`- [ ]`).

**Goal:** 무신사 단품 SP가 자기 색·정규사이즈 행만 갖도록 — 크롤 정규화·색스코프(A), 확장 색채움(B), 기존 폴루션 정리 마이그레이션(C, 백업+드라이런+soft-delete), 라이브 검증(D).

**Architecture:** upsert 정규화 + save_crawl_result 단품 색필터로 재발 차단 + dedup 엔드포인트로 기존정리. 데이터 무결성 최우선(soft-delete·백업·dry-run·승인).

**Tech Stack:** Flask/SQLAlchemy, pytest. 워크트리 `C:/dev/_wt_poll`(branch feature/musinsa-dan-pollution-fix), app root `프로그램/_시스템`. 배포=push origin/main→AWS(검증된 정상 배포).

**스펙**: `docs/superpowers/specs/2026-06-24-무신사단품-재고폴루션-근본수정-design.md`

**핵심 파일(조사 확정)**: `lemouton/sources/service.py`(upsert_source_option ~135, save_crawl_result ~449), `lemouton/sources/models.py`(SourceOption, BundleSourceUrl), `webapp/routes/api_pricing.py`(_persist_option_stocks ~142, _match_option_so ~95, crawl-result handler ~1080).
