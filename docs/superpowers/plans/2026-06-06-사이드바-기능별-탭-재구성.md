# 사이드바 기능별 탭 재구성 (1단계) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 사이드바를 기능 위주 6개 대분류로 재배치하고, 3개 항목을 네비게이션에서만 숨긴다(기능·데이터 불변).

**Architecture:** 사이드바는 `data/sidebar_layout.json`(라이브) + `api_sidebar._default_layout()`(리셋/최초부팅 기본값) 두 곳이 소스이며, `partials/sidebar.html`이 이를 순회 렌더한다. 본 작업은 `_default_layout()`을 6그룹으로 재작성하고(=계약), 같은 구조로 라이브 JSON을 재생성하며, 크롤링 가이드 자동 주입 대상을 새 크롤링 그룹으로 바꾼다. 페이지/라우트/active_key는 일절 건드리지 않으므로 템플릿·하이라이트는 자동으로 따라온다.

**Tech Stack:** Python 3 / Flask / pytest. 실행 디렉터리 = `프로그램/_시스템`.

from webapp.routes import api_sidebar


def _active_keys(layout) -> list[str]:
    """standalone + 모든 stage 항목의 active_key 를 순서대로 수집(중복 카운트용 list)."""
    keys = [it.get('active_key') for it in layout.get('standalone', [])]
    for st in layout.get('stages', []):
        keys += [it.get('active_key') for it in st.get('items', [])]

    layout = api_sidebar._default_layout()
    names = [st['name'] for st in layout['stages']]
    assert names == ['모음전 상품관리', '매핑 현황', '크롤링&업로드', '구매', '판매', '기타']
