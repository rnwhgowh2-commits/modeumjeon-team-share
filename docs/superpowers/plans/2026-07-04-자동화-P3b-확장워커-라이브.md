# 자동화 실행 엔진 P3b — 확장 워커 루프(실제 자동 크롤) 구현 플랜

> **For agentic workers:** Task 1(서버)만 subagent-driven-development(TDD). **Task 2(확장 JS)는 pytest 불가 → 라이브 브라우저 검증 필수**(`/live-browser-verify`). Steps use checkbox syntax.

**Goal:** 로컬 크롬 확장이 서버에 "지금 크롤할 번들"을 주기적으로 물어보고, **검증된 기존 크롤 로직으로 실제 자동 크롤**을 돌게 한다. 크롤=로컬 원칙 유지(서버는 목록만).

**핵심 설계(확장 내부 조사 결과):** 확장은 **URL이 아니라 모음전(번들) 코드 단위**로 크롤한다 — `mgrEnqueue({codes})` → `runQueueBG` → `crawlBundleAllBG(code)` → `/api/bundles/{code}/option-matrix` → 소싱처별 병렬 크롤(동시 최대 3·적응형·재시도·로그인 세션) → `/api/sources/crawl-result` → `crawl-finalize`. **이 검증된 흐름을 그대로 재사용**한다. 그래서:
- **서버(Task 1, TDD):** 마감(due)난 URL → 그 URL이 든 **번들 코드**로 매핑해 내주는 엔드포인트.
- **확장(Task 2, 라이브검증):** 그 엔드포인트를 폴링해 `mgrEnqueue({codes})` 만 호출하는 **얇은 루프**. 새 크롤 코드 없음(재사용).

**대상 워크트리:** `C:\dev\모음전 프로젝트\_wt_auto_engine` (브랜치 `feature/automation-engine`)

**스펙:** [`design.md`](../specs/2026-07-04-자동화-연속배수큐-실행엔진-design.md) §5.3

**전제(P1·P2·P3a):** `crawl_schedule.py` 의 `due_products`·`due_crawl_payload`·`base_crawl_interval_seconds`. `GlobalSettings.crawl_auto_enabled`. `BundleSourceUrl(bundle_source_urls: model_code, source_key, url, url_type)`. 확장 재사용 함수: `bgFetch`·`mgrEnqueue({codes})`·`crawlBundleAllBG`(background.js). 확장 version 0.7.5 / `MOUM_EXT_VERSION="0.7.4"`(background.js:13).

**⚠️ 조용한 누락 방지(최우선):** 마감난 `SourceProduct.url`(정규화됨, 트래킹 제거)과 `BundleSourceUrl.url`(등록값, 정규화 안 됐을 수 있음)을 매칭할 때 **양쪽을 같은 정규화 함수로 통과시켜** 비교한다. 정규화 불일치 시 due URL이 어떤 번들에도 안 걸려 **영원히 안 긁히는데 에러도 없음**(이 프로젝트 조용한실패 클래스). 크롤 파이프라인이 SourceProduct 를 만들 때 쓰는 URL 정규화 함수를 찾아 재사용할 것.

```python
from datetime import datetime

from lemouton.sources.crawl_schedule import due_bundle_codes
from lemouton.sources.models import SourceProduct
from lemouton.sourcing.models import Model, BundleSourceUrl
from lemouton.pricing.settings import get_or_init

NOW = datetime(2026, 7, 4, 12, 0, 0)


def _enable(db, on=True):
    s = get_or_init(db); s.crawl_auto_enabled = on
    s.crawl_interval_hours = 6; s.crawl_interval_minutes = 0; db.flush()


def _bundle_with_url(db, code, url):
    db.add(Model(model_code=code)); db.flush()
    db.add(BundleSourceUrl(model_code=code, source_key="musinsa", url=url,
                           url_type="단품")); db.flush()


def _due_sp(db, url):
    sp = SourceProduct(site="musinsa", url=url, crawl_weight=1,
                       no_change_streak=0, last_fetched_at=None)  # 미크롤=마감
    db.add(sp); db.flush(); return sp


def test_due_url_maps_to_its_bundle_code(db):
    _enable(db, on=True)
    url = "https://www.musinsa.com/products/123"
    _bundle_with_url(db, "M001", url)
    _due_sp(db, url)
    codes = due_bundle_codes(db, now=NOW)
    assert "M001" in codes


def test_disabled_returns_empty(db):
    _enable(db, on=False)
    url = "https://www.musinsa.com/products/123"
    _bundle_with_url(db, "M001", url)
    _due_sp(db, url)
    assert due_bundle_codes(db, now=NOW) == []


def test_not_due_url_excluded(db):
    from datetime import timedelta
    _enable(db, on=True)
    url = "https://www.musinsa.com/products/999"
    _bundle_with_url(db, "M999", url)
    sp = _due_sp(db, url)
    sp.last_fetched_at = NOW - timedelta(hours=1)  # 6h 미만 → 아직
    db.flush()
    assert "M999" not in due_bundle_codes(db, now=NOW)


def test_codes_deduped(db):
    # 한 번들에 같은 due URL 두 번 등록돼도 코드 1개
    _enable(db, on=True)
    url = "https://www.musinsa.com/products/1"
    db.add(Model(model_code="D1")); db.flush()
    db.add(BundleSourceUrl(model_code="D1", source_key="musinsa", url=url, url_type="단품"))
    db.add(BundleSourceUrl(model_code="D1", source_key="musinsa", url=url, url_type="색상모음전"))
    db.flush()
    _due_sp(db, url)
    codes = due_bundle_codes(db, now=NOW)
    assert codes.count("D1") == 1
```

```python
def due_bundle_codes(session, *, now) -> list:
    """마감난 URL이 걸린 모음전(번들) 코드 목록(중복 제거). 확장이 이걸 크롤한다.

    실행/정지가 꺼져 있으면 빈 목록. URL 매칭은 파이프라인과 같은 정규화 사용.
    """
    from lemouton.pricing.settings import get_or_init
    from lemouton.sourcing.models import BundleSourceUrl
    if not bool(get_or_init(session).crawl_auto_enabled):
        return []
    base = base_crawl_interval_seconds(session)
    due = due_products(session, base_interval_seconds=base, now=now)
    # <정규화 함수> = 사전조사에서 찾은 것. 여기선 _norm_url 로 표기.
    due_urls = {_norm_url(p.url) for p in due}
    if not due_urls:
        return []
    codes = []
    seen = set()
    for bsu in session.query(BundleSourceUrl).all():
        if _norm_url(bsu.url) in due_urls and bsu.model_code not in seen:
            seen.add(bsu.model_code)
            codes.append(bsu.model_code)
    return codes
```
`_norm_url` 은 실제 정규화 함수로 대체(import). SourceProduct.url 이 이미 정규화 저장이면 `_norm_url(p.url)` 은 멱등이어야 한다.

```python
@bp.route('/crawl/due-bundles')
def crawl_due_bundles():
    """[읽기전용] 로컬 크롤러가 폴링 → 지금 크롤할 모음전 코드 목록."""
    from datetime import datetime, timezone

    s = SessionLocal()
    try:
        now = datetime.now(timezone.utc).replace(tzinfo=None)
        enabled = bool(get_or_init(s).crawl_auto_enabled)
        codes = due_bundle_codes(s, now=now)
        return jsonify({"enabled": enabled, "count": len(codes), "codes": codes})
    finally:
        s.close()
```

```javascript
// 2026-07-04 자동화: 서버 due-bundles 폴링 → 기존 크롤 큐로 위임(재사용)
const MOUM_POLL_MS = 60000; // 1분
let _pollTimer = null;
async function moumAutoPollOnce() {
  try {
    const r = await bgFetch("/api/crawl/due-bundles").then((x) => x.json());
    if (r && r.enabled && Array.isArray(r.codes) && r.codes.length) {
      mgrEnqueue({ codes: r.codes });   // 기존 큐/동시성/재시도 그대로
    }
  } catch (e) { console.warn("[moum-auto-poll]", e); }
}
function moumAutoPollStart() {
  if (_pollTimer) return;
  moumAutoPollOnce();
  _pollTimer = setInterval(moumAutoPollOnce, MOUM_POLL_MS);
}
function moumAutoPollStop() {
  if (_pollTimer) { clearInterval(_pollTimer); _pollTimer = null; }
}
```
