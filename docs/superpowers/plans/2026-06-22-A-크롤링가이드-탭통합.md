# A · 크롤링 가이드 탭 통합 — 구현 플랜

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 데이터·코드 지도와 신규 소싱처 추가 탭을 하나로 통합 — 지도(1~6탭)=단일 진실원천, 7신규추가=순서도. CLAUDE가 지도만 보고 신규 소싱처를 추가할 수 있게.

**Architecture:** `how_to_add` 라우트·템플릿을 완전 제거하고, `map.html`을 6→7 하위탭으로 확장한다. 전 탭에 공통 디자인 규율(Pretendard 1종·4색·"핵심" 라벨·줄글·중앙정렬)을 적용. 정본 마크다운(`크롤링-가이드.md`)과 SOP(`신규-소싱처-추가-가이드.md`)를 멀티 URL·혜택 종합·재고 3상태·특이사항으로 동기화.

**Tech Stack:** Flask(Blueprint, Jinja2 템플릿), 순수 HTML/CSS/JS(빌드 없음), pytest(라우트·템플릿 내용 테스트), 라이브 브라우저 검증.

**기준 브랜치:** origin/main 에서 분기한 **새 워크트리**에서 작업(현 `feature/crawl-worker-queue`는 239커밋 stale). 끝나면 origin/main 머지.

**범위 밖(후속 플랜):** C·D 가격탭 혜택 편집·전파, B 크롤 게이트. 본 플랜에서 가격 탭(3)은 **읽기전용 표시**(소싱처·방식 배지·차감식)까지만. 편집 UI는 C·D 플랜.

**스펙:** [2026-06-22-크롤링가이드-통합개편-design.md](../specs/2026-06-22-크롤링가이드-통합개편-design.md) — 탭별 내용·소싱처 매핑은 스펙 §3·부록 A가 단일 진실원천.
