# 가격 설정 바 UI 재설계 — 구현 계획 (Part A)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** §3 "옵션 통합 관리" 화면의 "소싱처 일괄 정책" 바를, 크롤 버튼과 통합된 "가격 설정 박스"로 시각 재설계한다 (UI만 — 재계산 엔진은 Part B 별도 계획).

**Architecture:** 순수 프론트엔드 변경. Jinja 부분 템플릿 `_matrix_v3.html` 의 HTML 블록 2개(`global-actions-v2` 크롤 버튼 + `bulk-policy-bar` 정책 바)를 하나의 통합 박스로 교체하고, 스타일을 `toss.css` 에 추가한다. 기존 칩 동작 JS(`.bp-chip` 팝업 등)는 클래스·id 를 그대로 유지해 그대로 작동시키며, "마진 방식 ↔ 칩 단위 일치" 로직만 JS 에 소량 추가한다.

**Tech Stack:** Flask + Jinja2 템플릿, 순수 CSS, 바닐라 JS. 백엔드·DB 변경 없음.

**검증 방식:** 템플릿/CSS/JS 변경이라 pytest 단위 테스트가 적용되지 않는다. 각 태스크는 브라우저 실접속 확인으로 검증한다 (프로젝트 규칙 `/ui-verify`). 라이브 서버 URL 은 구현자가 `프로그램/_시스템` 에서 Flask 앱을 띄워 확인한다.

**범위 밖 (Part B):** `↻ 가격만 재계산` 버튼의 실제 계산 기능. 이 계획에서는 버튼을 시각적으로만 정리하고, 클릭 시 동작은 기존 stub(alert) 그대로 둔다.
