## §디자인 (확정 — 시안 v4/v2/v3에서 사용자 선택)

- **① URL별 계수** = 매트릭스 각 URL 행에 스텝퍼 **「− ×3 +」**(1~5, ×N 표기, 「배 자주」 문구 없음) + 값 아래 안내문구 **"🔄 전체 크롤 1회당 N회전 크롤됩니다"**. 저장 대상 = `SourceProduct.crawl_weight`.
- **③ 마켓 속도** = 오른쪽 「자동 전송」 카드에 표: **마켓 · 계정(API) · 1개당 초(직접 입력) · 시간당 개수(파생) · 켜짐**, 마켓별 합산행 **"↳ 총 스토어 업로드수 X개/1시간"**. 단위 = **1개당 초**(초 클수록 천천히=안 막힘), 시간당 개수 = `3600 ÷ 초`.
  - ⚠️ **백엔드 재설계 필요**: P4 `MarketUploadPolicy` 는 **마켓 단위**(market PK). 확정 디자인은 **계정(API) 단위**(한 마켓에 여러 계정, 각 제한 따로 → 합산). → **추후 논의**(사용자 명시). P5에서는 화면만, 저장은 계정모델 확정 후.
- **⑤ 실행중 + 실패 유형화** = 왼쪽 카드(링+로그) + 실패는 **유형 컬럼(좌) + 상세(우)**, 각 실패 **옵션 > URL**. 유형 = 🚫차단 · 🔑로그인 · 🧩옵션 못읽음 · ⏱응답지연 · ❔유형 외(분류 안 됨). (데이터 지도·§5 에러이력 근거)
  - ⚠️ **선행 필요**: 크롤 결과가 **실패 유형**을 담아야 함(현재 status ok/error + 메시지만). 유형 분류 로직(에러 메시지→유형) + 실행중 실시간 데이터 = P3b 라이브 후.

from lemouton.sources.crawl_schedule import set_crawl_weight

def test_weight_clamped_1_to_5(db):

    assert sp.crawl_weight == 1

- [ ] **Step 3: 구현** — `crawl_schedule.py`

```python
def set_crawl_weight(session, source_product_id: int, weight) -> int:
    """URL(SourceProduct)의 계수(1~5) 저장. 1~5로 클램프. 호출자가 commit."""
    from lemouton.sources.models import SourceProduct
    sp = session.get(SourceProduct, source_product_id)
    if sp is None:
        raise ValueError(f"source_product {source_product_id} 없음")
    sp.crawl_weight = max(1, min(5, int(weight)))
    session.flush()
    return sp.crawl_weight
```

- [ ] **Step 4: 라우트** — `webapp/routes/api.py`

```python
@bp.post('/sources/crawl-weight')
def set_source_crawl_weight():
    """URL 계수(1~5) 저장. body: {source_product_id, weight}."""
    from lemouton.sources.crawl_schedule import set_crawl_weight
    data = request.get_json(silent=True) or {}
    s = SessionLocal()
    try:
        w = set_crawl_weight(s, int(data.get('source_product_id')), data.get('weight'))
        s.commit()
        return jsonify({"ok": True, "weight": w})
    except ValueError as e:
        return jsonify({"ok": False, "error": str(e)}), 404
    finally:
        s.close()
```

## Task 2 (프론트·라이브검증): 매트릭스 계수 스텝퍼 + 안내문구

- 매트릭스 각 소싱처 URL 행에 「− ×3 +」 스텝퍼(1~5) + 아래 "🔄 전체 크롤 1회당 N회전 크롤됩니다"(N=계수). +/− 클릭 → `POST /api/sources/crawl-weight` 저장, 안내문구 N 갱신.
- 대상 파일 `webapp/templates/bundles/_matrix_v3.html`(소싱처 컬럼/행). ⚠️ 편집 함정 [[reference_flow_html_encoding_traps]](null문자·CRLF·PowerShell 한글깸 — Python io.open utf-8로 편집).
- 계수 저장에 필요한 `source_product_id` 를 셀에 실어야(현재 없으면 매트릭스 데이터에 추가).
- 라이브 검증: 스텝퍼 눌러 저장되는지(새로고침 유지) + 안내문구 숫자 갱신.
