## 데이터 집계 결정 (옵션 N → 카드 1행)
- **소(소싱처) = 세트 레벨**(마켓 무관, 행마다 동일): `소싱처명`(대표=옵션 최다 소싱처) · `재고수`=옵션 src_stock 합 · `최종매입가`=대표(최저) — **최종매입가는 무거우므로 카드엔 재고수+소싱처명만, 최종매입가는 행펼침(D3) 상세에서**(이미 detail-matrix가 계산).
- **판(판매처) = 채널 레벨**: `설정수`=matched 옵션 mkt_stock 합(쿠팡=미상 null) · `판매가`=대표(최빈/최저 mkt_price).
- **신호등(E2)**: 재고 = 판=0(또는 0포함) → 심각(빨강, alert market_soldout/both_zero) / 소≠판 합 → 주의(주황) / 일치 → 정상. 가격 = price_spike → 주의 / 판매가 없음 → 주의 / else 정상. 전송 = last_sent_at 있음 → 정상 / 미전송 → 주의(미동기화 not_synced 포함).
- **알림 O4**: 카드 제목 ⚠N 숫자 제거. 문제는 행 신호등 + 행펼침 상세로.
