# Phase 1 실전송 런북 — 스코프 원샷 도구로 스스·쿠팡 통제 라이브 검증

> 도구: `프로그램/_시스템/scripts/live_send_skus.py`. 기본 드라이런. 실전송은 3중 조건(--live + --i-understand-live-send + 서버키 `MOUM_LIVE_UPLOAD`).
> 원칙: 돈·계정 리스크. 지정 SKU만 1회, 매 배치 사용자 확인. 연속 스케줄러와 분리.

---

## 절차

### 1) (제어) 드라이런으로 대상 값 확인 — 실전송 아님
```
cd "프로그램/_시스템"
ENVIRONMENT=team-share-dev DISABLE_AUTH=1 python -m scripts.live_send_skus --skus <SKU1>,<SKU2>
```
- 출력의 마켓별 건수·가격/재고 변동을 실제 판매처(스스·쿠팡 셀러) 화면과 대조.
- 숫자가 화면과 100% 일치해야 다음 단계. 불일치면 멈추고 원인(매핑·마진·크롤 stale) 규명.

### 2) (사용자) 서버키 켜기 — 배포 행위
- 배포 env `MOUM_LIVE_UPLOAD=1` 설정 후 재배포. (env·배포는 사용자 직접. AI가 못 켬.)
- 켠 상태에서도 이 도구는 `--live --i-understand-live-send` 를 줘야만 실제 전송.
- ★연속 스케줄러 화면키(`autosend_mode`)는 여전히 `preview` 로 두어야 함(전량 자동전송 방지). 이 도구는 화면키와 분리라 무관.

### 3) (제어 + 사용자 확인) 소량 실전송
```
cd "프로그램/_시스템"
ENVIRONMENT=team-share-dev python -m scripts.live_send_skus --skus <SKU1> --live --i-understand-live-send
```
- "모드: ★실전송(LIVE)" 확인. 실패 시 DLQ(`data/uploader_dlq.jsonl`) — 거짓성공 아님(응답코드 판정).
- 먼저 옵션 1개로 시작.

### 4) (사용자) 라이브 셀러 화면 대조
- 스스·쿠팡 셀러오피스에서 그 옵션의 가격·재고가 의도대로 바뀌었는지 눈으로 확인.
- 이상 없으면 SKU 수를 조금씩 늘려 반복(3→4단계).

### 5) (사용자) 마무리
- 검증 끝나면 서버키 `MOUM_LIVE_UPLOAD` 다시 OFF(재배포). 연속 자동전송은 별도 결정 전까지 잠금 유지.

---

## 안전 체크리스트
- [ ] 드라이런 숫자 = 실제 셀러 화면 (1단계)
- [ ] 서버키는 사용자가 켬 (2단계)
- [ ] 연속 화면키 `autosend_mode='preview'` 유지 (전량 자동전송 차단)
- [ ] 옵션 1개부터, 실패는 DLQ로 표면화
- [ ] 검증 후 서버키 OFF 복귀
