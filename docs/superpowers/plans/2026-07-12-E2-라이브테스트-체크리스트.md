# E2 — 소싱처 주문상태 자동확인(로컬 크롬확장) 라이브 테스트 체크리스트

> 관계 선언: 프로젝트 문서 · Task E2 산출물 검증 · 상위 = CLAUDE.md 🔒3대원칙(크롤=로컬) · 대상 = 마진계산기(`/orders?tab=margin`) '✓ 확인' 버튼

## 왜 이 문서가 필요한가 (정직성)

E2 는 **크롬 확장 + 실제 소싱처 로그인**이 있어야만 끝단까지 동작한다. Python 테스트로는
검증 불가능한 영역(확장 서비스워커·페이지 DOM 파싱)이라, 아래를 **사용자가 직접 실행**해
확인해야 "동작한다"고 말할 수 있다. 자동 테스트로 검증된 것은 **간단메모 순수 파싱**(Node)과
**seam 무수정 이식**(margin_embed 동치 가드)뿐이다.

## 사전 준비 (필수)

1. **확장 재복사 + 재로드** — 실제 로드되는 확장은 데스크톱 사본이다(리포 파일 아님).
   - 리포 `프로그램/_시스템/extension/moum-crawler/` 전체를 데스크톱 확장 폴더로 덮어쓰기
   - `chrome://extensions` → 모음전 크롤러 → 새로고침(↻) → **버전 0.7.19** 표시 확인
2. mou-m.com(또는 localhost:5053) 로그인 → `/orders?tab=margin` 진입
   - ※ `ext_bridge.js` 는 서버 static(확장 아님) → 페이지 **강력 새로고침(Ctrl+Shift+R)** 로 최신 반영.
3. 개발자도구 콘솔에서 확장 감지 + 타입 메서드 노출 확인:
   ```js
   document.documentElement.getAttribute('data-moum-ext')   // "0.7.8" 이어야 함
   window.MoumExt && window.MoumExt.installed()             // true
   typeof window.MoumExt.checkSourcingOrder                 // "function" (미노출이면 브리지 구버전)
   ```
   > iframe(마진 임베드) 안에서 확인: `window.parent.MoumExt.installed()` /
   > `typeof window.parent.MoumExt.checkSourcingOrder === 'function'` (seam 은 부모의 MoumExt 를 쓴다)
4. **각 소싱처에 이 브라우저로 1회 로그인** (무신사·SSG·롯데온·SSF·르무통 등) — 확장은 별도
   프로필이 아니라 **현재 로그인된 이 브라우저 세션/쿠키**를 그대로 쓴다.

## 확장 미설치/미로그인 정직성 확인 (먼저)

| 상황 | 만드는 법 | 기대 결과(셀 표시) |
|---|---|---|
| 확장 미설치 | 확장 비활성화 후 '✓ 확인' | `실패: 모음전 크롬확장 필요 (로컬 크롤)…` (서버 호출 X) |
| 브리지 구버전 | `ext_bridge.js` 옛버전(캐시) | `실패: 모음전 확장/브리지 업데이트 필요…` (원시 TypeError 금지) |
| 소싱처 미로그인 | 해당 소싱처 로그아웃 후 '✓ 확인' | `실패: 로그인 필요 …` (거짓 '배송완료' 금지) |
| 지원 안 하는 소싱처 | 현대H몰/스마트스토어 메모 | `실패: 소싱처 식별 불가 — 확인 불가` |
| 배치 중지 | 일괄확인 중 '중지' 클릭 | 진행 중 확인 즉시 중단(⏹ fetch 취소) — **가짜 상태 안 씀** |
| 송장만 있고 상태문구 없음 | (드묾) | `확인불가` + 송장 노출 — **'배송중' 단정 금지**(배송완료 주문도 송장 있음) |

> 위 항목이 **조용히 성공으로 둔갑하지 않고** 사유가 셀에 뜨면 정직성 통과.

## 소싱처별 실행 절차 & 확인 항목

각 행: 마진계산기 분석 후, 상세표의 **'소싱처 주문상태'** 열에서 **`✓ 확인`** 클릭
(또는 `📦 소싱처 일괄확인`). 새 창이 열려 주문 URL 로 이동 → 상태 읽고 → 창 자동 닫힘.

**보고해 주세요(각 소싱처마다):** 실제 소싱처 화면의 주문상태 vs 셀에 뜬 값 / 택배사 / 송장번호.

| 소싱처 | 파서 근거 | 테스트할 메모(URL 형태) | 기대(status) | 반드시 보고 |
|---|---|---|---|---|
| 무신사 | 전용 셀렉터(`p.company-name`·`button.tracking-number`) + 범용 | `…무신사 / <계정> https://www.musinsa.com/order/order-detail/<주문번호>` | 배송완료/배송중/미발송 | 송장·택배사 잡히는지 (배송조회 버튼 클릭 흐름 **미이식** → 미표시 가능) |
| SSG | 범용(라벨+키워드) | `…SSG / <계정> https://pay.ssg.com/myssg/orderInfoDetail.ssg?orordNo=<주문번호>` | 배송완료/배송중 | 상태 정확도 |
| 롯데온 | 범용 | `…롯데온 / <계정> https://www.lotteon.com/…orderNo=<주문번호>` | 배송완료/배송중 | 롯데 DeliveryTrace 이동 **미이식** → 송장 미표시 가능 |
| SSF | 전용(`checkDelivery` onclick) | `…SSF / <계정> https://www.ssfshop.com/secured/mypage/<주문번호>/orderInfo` | 배송완료/배송중 | onclick 송장 파싱되는지 |
| 르무통 | 범용 | `…르무통 / <계정> https://lemouton.co.kr/…주문상세URL` | 상태 | 상태 라벨 잡히는지 |
| 현대H몰 | **미지원** | 아무 hmall URL | `소싱처 식별 불가` | 정직 실패 뜨는지(오탐 아님) |
| 스마트스토어 | **미지원** | 아무 smartstore URL | `소싱처 식별 불가` | 정직 실패 뜨는지 |

## 파서 신뢰도(솔직 고지)

- **높음**: 로그인 리다이렉트 감지, 상태 키워드 분류(배송완료/취소/반품/교환/배송중/미발송) — 원본
  `_classify_status_text` 우선순위 그대로.
- **중간**: 무신사·SSF 송장/택배사(전용 셀렉터는 이식했으나 원본의 '배송조회 버튼 클릭 후 재조회'
  흐름은 미이식). 범용 라벨 추출.
- **낮음/미검증**: 롯데온·SSG·르무통 송장(전용 DeliveryTrace/버튼 흐름 미이식 → 상태는 잡혀도
  송장은 빈 값 가능). 버튼/메뉴 텍스트로 인한 상태 오탐 가능성(경량 DOM 정제만 적용).
- **미지원**: 현대H몰·스마트스토어(원본 sourcing_checker 에 체커 없음) → 정직하게 '확인 불가'.

## 회귀/디버그 팁

- 확장 서비스워커 콘솔(`chrome://extensions` → 서비스 워커 검사)에서 `sourcing.check-order`
  요청·`[1/3]…[3/3]` 로그 확인.
- 응답 형태: `{ok, order_status, courier, tracking, site_name, source:'ext-local', logs[], error, is_logged_in}`.
- 상태가 계속 `확인불가` 면: (a) 로그인 여부 (b) 실제 페이지에 상태 텍스트가 있는지 (c) 파서가
  못 읽는 새 DOM 구조인지 — body innerText 스냅샷을 확인해 셀렉터/키워드 보강 필요.
