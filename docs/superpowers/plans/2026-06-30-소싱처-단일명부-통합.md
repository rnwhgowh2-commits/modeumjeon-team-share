# 소싱처 단일 명부 통합 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement task-by-task. Steps use checkbox (`- [ ]`).

**Goal:** 프로그램 등록 소싱처를 `source_key` 기준 단일 명부(SourcingSource)로 통합 — 이름=수정가능 껍데기, 로고=URL favicon, 모든 표면이 명부에서 읽기. 운영센터 탭 숨김.

**Architecture:** SourcingSource 를 전 소싱처(빌트인+커스텀) 단일 명부로 확장. 라벨/로고를 명부 단일원천화(하드코딩 5곳 제거). 사전=관리지점, 가이드/위젯/매트릭스=읽기. OptionSourceUrl FK·크롤 배관·SourceRegistry 물리삭제는 **안 건드림**(무결성 안전선).

**Tech Stack:** Flask, SQLAlchemy, SQLite(라이브)/Postgres(dev), Jinja, vanilla JS. pytest.

**Spec:** docs/superpowers/specs/2026-06-30-소싱처-단일명부-통합-design.md
