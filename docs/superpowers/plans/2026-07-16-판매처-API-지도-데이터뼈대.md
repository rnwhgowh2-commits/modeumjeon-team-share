# 판매처 API 지도 — 데이터 뼈대 (Plan 1) 구현 플랜

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 판매처 마켓 API를 담는 **단일 데이터 파일(JSON)** 과 그 무결성을 지키는 검증(스키마·완성게이트)·서빙 라우트를 만든다. 화면 재작성은 다루지 않는다(Plan 2).

**Architecture:** `MKT_CATS` 등 map.html 인라인 데이터를 → 서버에 있는 `marketplace_api_map.json` 한 파일로 이관·확장(각 API에 실제 요청/응답·필드뜻·성공판정·persistIds·fnKey·tabs 추가). 순수 Python 검증기(무의존)가 스키마와 "완성 게이트"를 강제한다. Flask 라우트가 이 JSON을 서빙(Plan 2 화면이 fetch). 쿠팡·스마트스토어를 본보기로 채운다.

**Tech Stack:** Python 3 / Flask (Blueprint) / pytest. 새 외부 의존 없음(표준 라이브러리 `json`만).

**Spec:** `docs/superpowers/specs/2026-07-16-판매처-마켓별-API-단일진실원천-design.md`

**⚠️ 범위 밖(Plan 2):** map.html의 7탭 재편·기능비교·상세 드로어 렌더는 **디자인 변경**이라 `/design-mockup` 게이트 후 별도 플랜. 이 플랜은 데이터·검증·서빙까지.

## File Structure

| 파일 | 책임 |
|---|---|
| `프로그램/_시스템/webapp/data/marketplace_api_map.json` | **단일 진실 원천** 데이터(markets·unifiedStatuses·transitions·codes·apis) |
| `프로그램/_시스템/webapp/marketplace_api_map.py` | 로더 + 순수 Python 검증기(스키마·완성게이트) |
| `프로그램/_시스템/webapp/routes/marketplace_guide.py` | (수정) `/marketplace-guide/map-data.json` 서빙 라우트 추가 |
| `프로그램/_시스템/tests/marketplace_guide/test_api_map_schema.py` | 스키마·완성게이트·라우트 테스트 |
| `docs/markets/*.yaml` | (삭제) 데이터 파일로 이관 후 제거 |

**데이터 스키마 (JSON 최상위 키):**
```
schema_version: 1
markets[]         {id,label,base_url,auth,env_prefixes[],concurrency}
unifiedStatuses[] {id,name,meaning,order}
transitions[]     {from,to,requires[],perMarket{market: apiId|"unsupported"}}
codes[]           {kind,market,code,label,meaning,unified,st}
apis[]            {id,market,fnKey,tabs[],category,nm,dir,st,endpoint,
                   req,res,fields[{key,meaning,example}],success,idTraps[],persistIds[],codeRef}
```
**완성 게이트 규칙:** `st ∈ {ok,code}` 인 api 는 `req·res·fields·success` 가 모두 비어있지 않아야 한다. `st ∈ {off,todo}` 는 골격만 허용.

```python
"""판매처 API 지도 데이터 로더 + 검증기 (무의존, 순수 Python).

단일 진실 원천: webapp/data/marketplace_api_map.json
- validate_map: 스키마 필수키 + 완성게이트(st∈{ok,code}→req·res·fields·success 필수)
  + api id 유일성 + transitions/perMarket 참조무결성.
"""
from __future__ import annotations
import json, os

_PATH = os.path.join(os.path.dirname(__file__), "data", "marketplace_api_map.json")

TOP_KEYS = ["schema_version", "markets", "unifiedStatuses", "transitions", "codes", "apis"]
API_KEYS = ["id","market","fnKey","tabs","category","nm","dir","st",
            "endpoint","req","res","fields","success","idTraps","persistIds","codeRef"]
GATE_STATUSES = {"ok", "code"}

def load_map(path: str = _PATH) -> dict:
    with open(path, encoding="utf-8") as f:
        return json.load(f)

def validate_map(data: dict) -> list[str]:
    errors: list[str] = []
    for k in TOP_KEYS:
        if k not in data:
            errors.append(f"최상위 키 누락: {k}")
    apis = data.get("apis", [])
    seen = set()
    for a in apis:
        aid = a.get("id", "<id없음>")
        if aid in seen:
            errors.append(f"api id 중복: {aid}")
        seen.add(aid)
        for k in API_KEYS:
            if k not in a:
                errors.append(f"api[{aid}] 필드 누락: {k}")
        if a.get("st") in GATE_STATUSES:
            for g in ("req", "res", "fields", "success"):
                if not a.get(g):
                    errors.append(f"완성게이트 위반 api[{aid}]: st={a.get('st')} 인데 {g} 비어있음")
    ids = {a.get("id") for a in apis}
    for t in data.get("transitions", []):
        for mk, ref in (t.get("perMarket") or {}).items():
            if ref != "unsupported" and ref not in ids:
                errors.append(f"transition {t.get('from')}→{t.get('to')} perMarket[{mk}] 참조 없음: {ref}")
    return errors
```

## Task 5: 쿠팡·스마트스토어 본보기 채우기 (기존 yaml·MKT_CATS → apis[])

기존 `docs/markets/coupang.yaml`·`smartstore.yaml` 과 map.html `MKT_CATS` 의 쿠팡/스스 항목을 새 `apis[]` 스키마로 이관한다. **이관 규칙**을 적용하고, 이미 연동된(st=ok/code) 작업은 완성게이트 4종을 채운다.

**이관 규칙 (yaml/MKT_CATS 1항목 → apis[] 1건):**
- `id` = `{market}.{fnKey}` (예: `coupang.product.get_detail`)
- `fnKey` = 마켓 공통 논리명(비교 축). 같은 일을 하는 스스/쿠팡 API 는 **같은 fnKey**.
- `tabs` = 스펙 §4 업무탭 중 해당(예: 상품 조회 → `["상품 조회"]`, 가격변경 → `["상품 관리"]`).
- `category`·`nm`·`dir`·`st`·`endpoint`·`code`(→codeRef) = MKT_CATS/yaml 그대로.
- `req`·`res`·`fields`·`success` = yaml `fields`/`response` + 실제 응답(있으면). **없으면 st 를 off 로 낮춰 미완 처리(거짓 완성 금지).**
- `idTraps`·`persistIds` = 조회 API 면 채움(예: 쿠팡 `vendorItemId` 는 등록 응답에 없음 → persistIds 로 저장 필요).
