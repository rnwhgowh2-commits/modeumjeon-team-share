# 송장 작업 화면 구조 — 설계서

> **작성** 2026-07-24 · **상태** 사장님 확정 (시안 v1~v4 4회 반복 후)
> **관계** 주문 내역(`/orders/?tab=list`) · 배송검사(`/orders/?tab=inspect`) 재편
> **시안** 바탕화면 `모음전 시안 v4 분류표시 10안.html`

---

## 1. 왜 하는가

사장님 발화: *"지금 너무 UI/UX 복잡해서 사용하기 불편함. 위상이 구분이 안 가서 기능의 목적 및 어디서 무엇할지 느낌이 하나도 안 옴."*

원인은 기능 부재가 아니라 **배치**였다. 코드 조사 결과:

- 송장 관련 기능 3종(송장 전송 · 중복 검사 · 배송흐름 검사)이 **모두 이미 구현돼 있다**
- 그중 배송검사는 **화면이 두 벌**이다 — 주문 내역 안(`tab=list`의 「더망고 대조」 카드 + 「🚚 배송검사」 뷰 탭)과 사이드바 별도 탭(`tab=inspect`). 더망고 업로드 창도 두 개. 데이터는 `/orders/inspect/*` 로 **같은 곳**을 본다 → 순수 화면 문제
- 주문 내역 한 화면에 성격이 다른 덩어리 4개가 평평하게 쌓여 있다: 더망고 업로드 카드 / 뷰 탭 3개 / 왼쪽 배송검사 막대 / 표 아래 송장 전송 띠
- 즉 **입력(업로드) · 판정(검사) · 실행(전송)** 이 한 평면에 섞여 있다

사장님 실제 작업 순서(확인): **중복 검사 → 송장 전송 → (마지막) 배송흐름 검사.** 한 줄로 이어지는 한 작업이다.
배송검사 화면은 두 벌 다 **"거의 안 썼다"** — 기능이 없어서가 아니라 어디서 뭘 하는지 몰라서다.

## 2. 무엇을 만드는가

### 2.1 사이드바

```
판매
 ├ 📋 주문 내역        ← 조회 전용으로 비움
 ├ 📦 송장 작업        ← 신설
 ├ 🚚 배송검사         ← 삭제 (②·④로 흡수)
 ├ 💬 CS
 ├ 🆕 신규 상품 등록
 ├ 🧮 마진 계산기
 └ 🔍 샵마인 대조
```

### 2.2 「📦 송장 작업」 — 3단 배치

```
┌ 왼쪽 250px ─┐ ┌ 가운데 ────────────────┐ ┌ 오른쪽 292px ┐
│ ① 더망고     │ │ (고른 단계의 작업창)     │ │ 🧾 배송흐름   │
│ ② 걸러내기   │ │                        │ │    감시       │
│ ③ 전송       │ │ ─────────────────      │ │  22건 멈춤    │
│ ④ 검산       │ │ ✓ 이 단계만 돌린 결과   │ │  [모아보기 →] │
│              │ │   (별도 카드)           │ │  마켓별 건수  │
└──────────────┘ └────────────────────────┘ └──────────────┘
```

- **왼쪽** — 네 단계 목록. 각 항목에 **「이 단계만」** 버튼(그 단계 하나만 실행)
- **가운데** — 고른 단계의 작업창. 따로 돌린 경우 **결과가 별도 카드로 아래에 열린다**(D2안)
- **오른쪽** — 배송흐름 감시. **항상** 떠 있다

### 2.3 네 단계

| | 이름 | 하는 일 | 엑셀 |
|---|---|---|---|
| ① | 더망고 엑셀 올리기 | 오픈마켓주문번호로 마켓 주문과 대조 | 더망고 |
| ② | 중복·문제 걸러내기 | 이미 송장 들어간 건·문제 건 분류 | (①의 결과) |
| ③ | 택배사 엑셀 올려 전송 | 깨끗한 것만 마켓으로 송장 전송 | 택배사 |
| ④ | 배송흐름 검산 | **송장 넣고 24시간 넘게 배송 흐름이 안 바뀐 주문** | **없음** |

**★ ④는 엑셀과 무관하다.** 사장님 정정: *"배송검사 목적은 송장번호 입력한 상품 중 24시간 이내 배송흐름 변화가 없는 주문건 보여주는 거야. 엑셀 업로드와 상관없음."*
→ 상시 감시. 아무 때나 열 수 있고, 왼쪽 목록·오른쪽 감시 칸 어디서든 진입한다.

### 2.4 ④ 모아보기 (G1안)

- 오른쪽 감시 칸의 「모아보기 →」 또는 왼쪽 ④ 클릭 → **가운데가 그 목록으로 바뀌고** 「← 송장 작업으로 돌아가기」
- 표 **맨 앞 칸 = 「멈춘 지」**(경과 시간). 24시간 넘으면 색(amber)
- 기준은 **24시간 고정**. (12·48시간 선택은 후속 — 사장님이 원하면)

### 2.5 분류 표시 = H5 「작은 점 하나만 색」

②의 다섯 분류를 **배경 색칠 없이** 표시한다. 글자 앞 7px 점만 색.

| 분류 | 점 색 | 건수 예 |
|---|---|---|
| 발송 대상 · 마켓에 송장 없음 | 파랑 `--blue` | 50 |
| 배송흐름 정체 · 송장 넣었는데 안 넘어감 | 노랑 `#EAB308` | 22 |
| 확인불가 · 매칭 안 됨 | 빨강 `--red` | 15 |
| 취소·반품·교환 | 회색 `#D8DDE3` | 137 |
| 이미 발송·완료 | 회색 `#D8DDE3` | 198 |

행 구조: `점 · 이름 · (회색 부연) ······ 건수(우측·자릿수 정렬)`, 행 사이 1px 실선.

### 2.6 「📋 주문 내역」 (C1 + F4)

- 송장 관련 부품 **전부 제거** — 더망고 카드 · 뷰 탭 3개 · 왼쪽 배송검사 막대 · 표 아래 송장 전송 띠
- 왼쪽 칸 **맨 위 = 「모든 항목에서 찾기」**. 칸 폭을 넘지 않게(`box-sizing:border-box`)
- 남는 것: 찾기 / 마켓 / 기간 / 엑셀 양식 / 표

### 2.7 열 필터 — 모든 표의 모든 열

지금 주문 내역 표에만 있는 필터(`th.fcol` 클릭 → `#colpop`)를 **모든 표**에 붙인다. 동작·모양은 지금과 동일(찾기칸 · (모두 선택) · 값별 건수 · 취소/확인).
대상: 주문 내역 표 / ② 분류별 표 / ④ 배송흐름 표.

### 2.8 찾기 범위 확대

지금은 `상품명 + 수령자` **두 개만** 본다 — [`orders/index.html:647`](../../../프로그램/_시스템/webapp/templates/orders/index.html)
```js
if(srch){var s=srch.toLowerCase();if(((r['상품명']||'')+ (r['수령자']||'')).toLowerCase().indexOf(s)<0)return false;}
```
→ **불러온 모든 항목(29개 열 전부)** 에서 찾도록 바꾼다. 라벨도 「검색 (상품·수령자)」 → 「모든 항목에서 찾기」.

## 4. 데이터 · 서버

### 4.1 옮기는 것 없음

②는 지금과 같이 `/orders/inspect/*`(더망고 적재분)를 읽는다. 화면만 재배치이므로 **DB 이사·마이그레이션 없음.**

### 4.2 ④ 배송흐름 감시 = 신규 계산

**정의** — 다음을 모두 만족하는 주문 라인:
1. 송장번호가 있다 (`송장입력` 비어있지 않음)
2. 주문상태가 아직 **배송중 이후로 넘어가지 않았다** (결제완료·배송준비중)
3. 송장이 처음 확인된 시각으로부터 **24시간 초과** 경과

**경과 시각의 원천** — 송장 원장(`lemouton/markets/invoice_ledger`)이 이미 "한 번 본 송장"을 보관한다. 여기에 **처음 본 시각**을 기준으로 경과를 계산한다. 원장에 시각 컬럼이 없으면 추가한다.
★ 값을 지어내지 않는다 — 처음 본 시각을 모르는 건은 「확인 불가」로 두고 감시 대상에서 뺀다(조용한 오탐 금지).

**표시** — 「멈춘 지」 = 지금 − 처음 본 시각. 24시간 초과면 amber.

### 4.3 라우트

| 경로 | 용도 |
|---|---|
| `GET /orders/?tab=ship` | 송장 작업 화면 (기존 subtab 구조를 그대로 따른다) |
| `GET /orders/flow-stall.json` | ④ 목록 (신규) |
| 기존 `/orders/inspect/*` | ① ② 그대로 |
| 기존 `/orders/invoice/upload`·`/send` | ③ 그대로 |

`tab=inspect` 는 `tab=ship` 으로 **영구 이동(301)**. 북마크가 깨지지 않게 한다.

## 5. 나누어 올리는 순서

한 번에 다 올리지 않는다. 각 단계가 **혼자서도 말이 되는** 크기다.

| 단계 | 내용 | 되돌리기 |
|---|---|---|
| **1** | 3장 고치기 — 정렬 규칙(A) · 찾기 범위 확대 · 찾기 칸 폭(D) | 쉬움 |
| **2** | 「📦 송장 작업」 탭 신설 + ①②③ 이사 (주문 내역은 아직 안 건드림) | 탭만 숨기면 됨 |
| **3** | 주문 내역 비우기 + 「🚚 배송검사」 탭 제거·이동 | 되돌리려면 되살려야 함 |
| **4** | ④ 배송흐름 감시(24시간) 신규 + 모든 표에 열 필터 | 새 기능이라 독립 |

## 7. 검증 기준

1. 「주문 내역」에 송장 관련 부품이 **하나도 없다**
2. 「배송검사」 탭이 사라지고, 더망고 올리는 곳이 **한 군데**다
3. 네 단계를 **순서대로도, 하나만 따로도** 돌릴 수 있고 결과가 따로 나온다
4. ④가 **엑셀을 올리지 않아도** 값을 보여준다
5. **모든 표의 모든 열** 제목을 누르면 필터가 열린다
6. 찾기에 **송장번호·주문번호·계정·주소**를 넣어도 결과가 나온다
7. 상태·배지 열이 **가운데** 정렬이다
8. 라이브 URL에서 위 7가지를 실제로 확인한다
