## 0. 왜 이 문서가 뒤늦게 생겼나

v1 → A안 v2 → v3 → v4 → v5 → **v6 확정본** 까지 6벌을 사장님과 함께 골랐는데,
**HTML 시안만 바탕화면에 남고 설계서로 옮겨지지 않았습니다.**

이 세션에서 실제로 사고가 났습니다 — 크롤 주기 설계서가 워크트리와 함께 사라졌고,
시안 7 도 없어졌습니다. **HTML 은 흩어집니다.** 그래서 문서로 고정합니다.

---

## 1. 이 화면이 풀려는 문제

> 더망고 대비 우위 — **주문 한 줄에 매입가와 판매가가 같이 보여 실마진이 즉시 보인다.**

기존 `/orders` 는 「판 것」만 보여줍니다. 얼마에 사 오는지는 다른 화면에 있습니다.
그래서 **역마진 주문을 눈으로 못 잡습니다.** 이 화면은 그걸 한 줄에 붙입니다.

## 2. 목록 — 9열

```
☑  ▸  [주문]        [상품]         [매입 → 판매]  [남는 돈]  [주문상태]  [송장]   [주문일시]
26  24   210px        가변            200px         96px       96px      126px     110px
```

| 열 | 담기는 것 |
|---|---|
| **☑** | 일괄 선택 |
| **▸** | 펼침 토글 (§3) |
| **주문** | 마켓 배지 + 계정 + **등록경로 배지** + 주문번호 + 플래그 |
| **상품** | **76px 정사각 이미지** + 소싱처 칩 + 브랜드 + 상품명 + 옵션 + 링크 3종 |
| **매입 → 판매** | 좌우 대비. 매입가가 바뀌었으면 **옛 가격을 취소선**으로 위에 |
| **남는 돈** | 마진 금액(16px 굵게) + 마진율. 위험 색상 |
| **주문상태** | 결제완료 / 발송준비 / 발송완료 |
| **송장** | 「송장 전송」 버튼 또는 송장번호 |
| **주문일시** | 날짜(굵게) + 시각(작게, 아래) |

### 2-1. 주문 칸 상세

```
[11번가] rnwhgowh3 [대량등록]
20260717085581933
[매입 ▲1,546]
```

- **마켓 배지** — 마켓 고유색. 11번가 `#F43142` · 롯데ON `#DA291C` · 스마트스토어 `#03C75A`
- **계정** — 다계정이므로 어느 계정 주문인지 필수
- **등록경로 배지** — `모음전`(회색) / `대량등록`(파랑). `ProductDraft.origin` 근거
- **플래그** — `매입 ▲1,546`(가격 변동) · `S`(품절) · `P`(부분)

### 2-2. 상품 칸 상세

```
[사진 76×76]  SSG.com 나이키
              [르무통] 나이키 여성 에센셜 우븐 하이웨이스트 카고 쇼츠
              HM6983-010 · 070 · 1개
              [마켓 ↗] [소싱처 ↗] [상품관리 ↗]
```

- 이미지는 **정사각 76px + 테두리** — 사진이 없어도 들어갈 자리가 보이게 (v6 에서 48→76 확대)
- **소싱처 칩 + 브랜드**를 상품명 위에 — 어디서 사 오는지가 상품명보다 먼저
- 링크 3종으로 마켓·소싱처·상품관리로 바로 이동

### 2-3. 매입 → 판매

```
  31,685        →     47,900
  33,231
```

- 왼쪽 = 최종매입가, 오른쪽 = 판매가
- **가격이 바뀌었으면 옛 값을 취소선**으로 위에 겹쳐 표기 (`31,685` 취소선 → `33,231` 현재)
- 주문 시점과 지금이 다를 수 있으므로 **둘 다 보여야** 합니다 (2026-07-19 M4 결정과 동일 원칙)

### 2-4. 남는 돈 — 색으로 위험을 알림

| 마진율 | 색 |
|---|---|
| 정상 | 기본 |
| 주의 | 주황 `--warn` |
| **역마진 · 위험** | **빨강 `--bad`** |

> 시안 예시: 8,442원 / 17.6% = 주황, 762원 / 1.5% = 빨강.
> ⚠️ **경계값은 미확정** — 몇 % 부터 주황·빨강인지 정해야 합니다(§6).

### 2-5. 정렬 규율 (v6 에서 고친 것)

- **모든 셀 세로 가운데 정렬** — 윗정렬이었던 걸 가로 중앙선 기준으로. 상품 칸도 이미지와 글이 중앙선에 맞음
- 숫자는 `tabular-nums` 고정폭
- 「전체 복사 (발주용)」은 **파란 버튼 → 일반 버튼** (너무 눈에 띄어서 낮춤)

## 3. 펼침 — 좌우 2단 · 번호 스텝

행을 펼치면 **사오는 쪽과 파는 쪽**이 좌우로 갈립니다. 테두리 색으로 구분합니다.

```
🛒 소싱처 — 사오는 쪽 (주황 #FDBA74)      🏬 판매처 — 파는 쪽 (파랑 #93C5FD)
```

### 3-1. 왼쪽 — 소싱처 (3스텝)

**① 살 수 있나 확인**

| | |
|---|---|
| 원문 상품명 | 소싱처의 원래 이름 (우리가 가공하기 전) |
| 현재 재고 | 있음 / 없음 — 초록·빨강 |
| **현재가 (실시간)** | 33,231 `▲1,546` |
| **최종 매입가** | 혜택 다 반영된 실매입가 |

버튼 — `소싱처 ↗` · `가격·재고 이력` · `타소싱처 ↗`

> `타소싱처 ↗` 가 중요합니다 — 이 소싱처가 품절이면 **다른 데서 살 수 있는지** 바로 봅니다.

**② 얼마 남나 계산** — 영수증처럼 한 줄씩

```
   판매가                47,900
 − 마켓 수수료 13%       −6,227
 − 까대기                    −0
 − 최종 매입가          −33,231
 ─────────────────────────────
   남는 돈 (17.6%)        8,442
   정산예정금액          41,673
```

> **마진율은 최종매입가 기준** (2026-07-19 사장님 확정).

**③ 사고 나서 입력** — 실제 매입 후 기록

| 칸 | |
|---|---|
| 소싱처 주문번호 | `SSG-20260717-0091` |
| 실 구입금액 | 예상 매입가와 다를 수 있음 |
| 택배사 | |
| 송장번호 | **크롤로 자동** 채움 목표 |

> 까대기 건은 **엑셀 업로드**로 일괄 입력.

### 3-2. 오른쪽 — 판매처 (2스텝)

**① 어디로 보내나** — 각 항목에 `복사` 버튼

수령인 · 연락처 · 주소 · 배송 메시지 · **출고 기한**(D-1 은 빨강) · 이중 송장
→ 하단에 `전체 복사 (발주용)` — 소싱처 발주 폼에 한 번에 붙여넣기용

**② 지금 상태**

주문상태 · **마켓 원문 상태** · 주문상품번호 · 클레임 · CS 이력 · 담당자
→ `주문상태 변경 ▾`

> **마켓 원문 상태를 따로 두는 이유** — 우리 통일 상태와 마켓 원문이 다를 수 있어,
> 통일 과정에서 뭉개진 정보를 확인할 수 있어야 합니다.
> (`project_marketplace_order_status_unification` 참조)

## 4. 데이터 근거

| 화면 값 | 출처 |
|---|---|
| 등록경로 배지 | `ProductDraft.origin` → 마켓 상품ID 로 주문에 조인 |
| 최종 매입가 | `compute_final_price` (혜택 전부 반영) |
| 현재가 · 재고 | 최신 크롤 스냅샷 |
| 매입가 변동 (`▲1,546`) | 주문 시점 스냅샷 vs 현재 — `lemouton/orders/price_diff.py` |
| 마켓 수수료 | ⑧설정 탭 마켓별 수수료율 |
| 까대기 | 추가비용 템플릿 |
| 정산예정금액 | 마켓 정산 API |

> **저장소를 나누지 않습니다.** 주문은 모음전으로 올렸든 대량등록으로 올렸든 같은 마켓에서 옵니다.
> 「등록경로」 필터로 구분해 양쪽 모드에서 같은 데이터를 봅니다. (가공템플릿 설계서 §3-2 확정 원칙)

---

## 5. 화면 공유

- **모음전 모드 ⑤ 와 대량등록 모드 ⑤ 는 같은 화면 한 벌**을 마운트합니다.
- 차이는 **기본 필터**뿐 — 대량등록 모드는 `origin=대량등록` 이 기본으로 걸립니다.
- 화면·데이터 2벌 금지 (프로젝트 최상위 원칙: 중복·모순 금지).

## 6. ❓ v6 에 아직 없는 것

시안은 **표 본체만** 그렸습니다. 아래는 구현 전에 정해야 합니다.

| | 내용 |
|---|---|
| ❓ | **필터·검색 줄** — 가공템플릿 설계서 §5 가 「강력한 필터」를 필수로 적었는데 v6 에 없음.<br>마켓 · 계정 · 등록경로 · 소싱처 · **브랜드** · 주문상태 · 기간 · 역마진만 |
| ❓ | **마진 색 경계값** — 몇 % 부터 주황·빨강인지 (§2-4) |
| ❓ | **일괄 작업 줄** — ☑ 로 여러 건 고른 뒤 뭘 하나 (송장 일괄 전송 · 발주용 일괄 복사 · 상태 일괄 변경) |
| ❓ | **정렬** — 기본 정렬 기준. 주문일시 역순? 마진 낮은 순? |
| ❓ | **페이지 나눔** — 무한 스크롤 / 페이지 버튼 |

> **브랜드 필터**는 판매처 관리 재설계와 얽힙니다 —
> 주문 내역에는 브랜드가 안 실려 오므로 `ProductDraft` 와 마켓 상품ID 로 조인해야 합니다.
> 우리가 안 올린 상품(수기 등록·과거 상품)은 브랜드를 못 붙입니다. 별도 설계 필요.
