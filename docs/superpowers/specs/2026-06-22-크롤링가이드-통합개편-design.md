## 1. 배경·목표

소싱처 크롤링 가이드의 두 탭 — **데이터·코드 지도**(6 하위탭, 풍부) 와 **신규 소싱처 추가 가이드**(순서도 1장) — 가 분리돼 있고, 내용이 중복될 위험이 있다. 동시에 가격 혜택은 화면에서 **수정할 수 없고**, 크롤이 문서화된 로직을 **실제로 따르는지 보장이 약하다**.

**목표 3가지**
1. **A — 두 탭 통합**: 지도 양식을 기준으로 신규추가 절차를 흡수. CLAUDE가 **지도 한 곳만 읽고** 신규 소싱처를 추가할 수 있어야 한다.
2. **C·D — 혜택 편집·전파**: 가격 탭에서 소싱처별 혜택(명·%)을 수정 → fx → 셀·매트릭스 보기·로그 → 최종매입가까지 일관 반영.
3. **B — 전체크롤 강건화**: 전체 크롤이 가이드(1~6탭)에 적힌 로직을 **정확히 따르도록** 검증·실패표면화.

**범위·순서**: 설계는 셋 다 다루되, 구현은 **A → C·D → B** 순. 각 단계 라이브 검증 후 다음.

## 3. A — 탭 통합 설계

### 3.1 구조 — 지도 = 단일 진실원천, 7신규추가 = 순서도

- 상단 서브탭에서 **"신규 소싱처 추가"를 제거**하고 데이터·코드 지도로 흡수.
- 지도 보기 모드의 하위탭을 **6 → 7개**로: `1 전체흐름 · 2 재고 · 3 가격 · 4 매트릭스 로그 · 5 원칙 · 6 재발 · ➕7 신규추가`.
- **역할 분리 (핵심 원칙)**:
  - **1~6탭 = 참고자료(정답 카드)** — *무엇을*: 각 소싱처가 어떻게 수집·계산하는지, 코드 `파일:라인`.
  - **7탭 = 절차(순서도)** — *어떤 순서로*: CLAUDE가 위→아래로 읽으며 **각 단계가 "N번 탭 보고 가장 비슷한 기존 소싱처 복제"** 라고 가리킴.
- 효과: ① 내용 중복 없음(단일 진실원천) ② CLAUDE는 7탭만 읽으면 각 단계의 참고 탭·소싱처를 앎 ③ 1~6탭을 고치면 절차는 자동으로 최신 참고.
- `보기 / 원문(MD)` 토글 유지. 원문 = 정본 `docs/크롤링-가이드.md`(`/sourcing-guide/map.md`).

### 3.2 탭별 레이아웃 (목업 합의 확정)

| 탭 | 레이아웃 | 내용 |
|---|---|---|
| **1 전체흐름** | 가로 카드(라벨 스택) | 4단계(크롤→저장→계산→표시), 각 `코드`·`위치`·`뜻` 라벨. ⚠️ BG_PARSE는 price·stock만 저장(혜택 미저장) |
| **2 재고** | 소싱처 카드(좌 76px 중앙정렬) | §3.3 |

| **4 매트릭스 로그** | 데이터4층 + 셀/매트릭스/로그 표 + ⭐완전한 B | window.DATA 공유, fx 후보별 키, 셀=계산식 동일 후보 |
| **5 원칙** | 좌측 제목칸 카드 | 폴백가 금지 · 크롤실패=표기 · 단일진실원천 · 누락 경고 |

| **7 신규추가** | 순서도(스텝 카드) | §3.4 |

### 3.3 재고 탭(2) — 데이터별 수집 + 3상태 판정 + 특이사항

상단에 **📖 용어 사전**(고정): `API 호출`(내부 요청 JSON, 가장 정확·선호) / `HTML 파싱`(원본 화면 문서) / `DOM 읽기`(브라우저가 그린 살아있는 화면, JS 렌더값 포함). **DOM·HTML = "화면 읽기" 계열(=API 아님)**.

소싱처 카드마다:
1. **축 라벨**: `로그인`(필요/불필요) · `크롤 위치`(크롬 확장/서버).
2. **데이터별 수집 방법(7항목)**: 상품명·대표이미지·추가이미지·옵션·재고·표면노출가·혜택 각각 `API`(파랑)/`HTML`/`DOM`/`템플릿`. → "어떤 데이터가 API로 되고 안 되는지" 한눈에(개선 우선순위).
3. **재고 3상태 판정 표** (이 탭의 핵심):
   - **품절(없음)** → 신호: 무신사 `outOfStock`→0 / SSG `usablInvQty=0` / SSF `statcd=SLDOUT` / 롯데온 매핑품절·`li.soldout` / 스스 `stockQuantity=0` / 르무통 `is_selling=F·재고0`
   - **한정(N개)** → 신호: 무신사 "잔여·마지막 N개"→실수량 / SSG `usablInvQty=N` / SSF `품절임박 ( N )`(괄호공백 주의) / 스스 `stockQuantity=N` / 르무통 option_stock_data 수량 / **롯데온 = 수량 미제공(못 잡음)**
   - **충분(표식없음)** → 처리: 표식 없음→999(충분) / **롯데온 = unknown(수량 미상)** (가짜 충분 금지)
   - **⚠ 특이사항**: URL에 재고 매핑했으나 **옵션없음**(매핑 실패·옵션 li 없음) → "옵션없음" / **크롤실패**(API·DOM 둘 다 실패, 딜페이지 파싱 실패, WAF 차단) → "크롤실패"(옛값 금지). 스스 **per-SKU 저장 누락 시 전부 999 둔갑** 명시.

> 소싱처별 매핑은 §부록 A 표. 코드 단일 진실원천 = 각 `crawlers/*.py` + `crawl_guide.py`의 `stock_rules`(soldout_markers·qty_patterns·no_marker_means).

### 3.4 신규추가 탭(7) — 백지 재설계 순서도

핵심 변화 ① **샘플 URL 1개 → "변형을 덮는 세트(3~6개)"** ② **혜택 종합 단계 신설**.

**왜 URL 여러 개?** 한 상품엔 그 상품 혜택·재고상태만 보임 → 변형을 덮는 세트를 받아 종합해야 전체 혜택 카탈로그+재고 3상태가 완성. 권장 커버리지 3축:
- **혜택 변형**(세일/쿠폰/후기적립은 상품마다 켜짐꺼짐) → 상시 vs 조건부 구분 + 트리거 키워드
- **재고 3상태**(품절·한정·충분 각 1개) → 한정수량 둔갑 차단
- **옵션 구조**(단일색·멀티색·딜) → 단일색 파싱·딜 매핑 버그 차단

**6단계** (각 단계가 1~6탭을 가리킴):
1. 샘플 URL 세트 받기 [사용자] — 대표 1개 + 3축 변형 2~5개
2. 수집 방식 4분면 분류 [CLAUDE] → **1 전체흐름** (API·HTML × 인증·비인증)
3. 7항목 위치 추출(재고 3단계 규칙 포함) [CLAUDE] → **2 재고** + **3 가격**
4. **혜택 종합 — 여러 URL 교차분석** [CLAUDE·신설] → **3 가격**. 모든 URL=상시(always) / 일부=조건부(conditional)+트리거 키워드 추출. 합집합=전체 혜택 카탈로그. (카드 `status`·`triggers`·`match` 채움)
5. 매입가 순차차감 로직 [CLAUDE] → **3 가격** (표면 − Σ혜택, 직전 잔액 기준)
6. 검증 크롤·대조 → 정답 카드 저장 [CLAUDE] → **5 원칙**·**6 재발** + 18항목 체크리스트 · `crawl_guide` JSON. **URL마다** 100% 일치할 때만 저장.

정본 SOP `docs/신규-소싱처-추가-가이드.md` 도 이 멀티 URL·혜택 종합 절차로 동기화한다(STEP 1 "URL 1개" → "URL 세트", STEP 4 앞에 "혜택 종합" 삽입).

## 4. C·D — 가격 탭 혜택 편집·전파 설계

### 4.1 정적/동적 혜택 + 반영 방식 (요구 4의 제안)

가격 탭의 소싱처별 혜택을 표로 보여주고 **오른쪽에서 값(명·%) 수정**. 각 혜택은 두 속성을 가진다:

- **유형**
  - **동적(크롤)** = 크롤이 상품마다 읽어오는 혜택(`dynamic_benefits_json`). 화면에선 **크롤값 읽기전용**.
  - **정적(고정)** = 사용자가 정하는 소싱처 공통 혜택(`SourceBenefitTemplate` / `crawl_guide.pricing.benefits[].value`). **파란 입력칸으로 직접 편집**(예: 현대카드 2.73%, 기프트 10%).
- **반영 방식**(고정형혜택 반영 방식 구분 = 요구 4 제안):
  - `크롤값 사용` — 크롤에서 읽은 %를 그대로.
  - `고정값 사용` — 크롤이 못 읽거나(현대카드 등) 항상 고정인 혜택은 사용자 설정 %를.
  - `크롤 우선·없으면 고정` — 크롤값 있으면 그걸, 없으면 고정값(폴백). **단, 폴백가 금지 원칙과 충돌 않게: 이 "고정"은 혜택 %의 기본값이지 표면가/최종가의 대표가 폴백이 아님**(가격 자체가 없으면 여전히 "가격없음").

데이터 모델: 기존 `crawl_guide.pricing.benefits[]`의 `value`·`method`·`base`·`status` 재사용 + **신규 필드 `value_mode ∈ {crawl, fixed, crawl_or_fixed}`** 추가(JSON 키라 마이그레이션 불필요, `validate_guide` 화이트리스트에 추가).

### 4.2 전파 경로 (수정 → 최종매입가)

```
혜택 수정·저장
  → 소싱처 기본셋팅(SourceBenefitTemplate) 반영  [sync_templates_from_crawl_guide]
  → compute_market_price / compute_breakdown 재계산  [fx]
  → window.DATA · SM_BREAKDOWNS 갱신  [loadMatrix → /option-matrix]
  → 셀 · 매트릭스 보기 · 로그 · 최종매입가  일제히 갱신
```

- 저장 엔드포인트 = 기존 `PUT /sourcing-guide/api/<sid>` 재사용(이미 `sync_templates_from_crawl_guide` + 선택적 `snapshot_bundle_from_templates` 보유). 가격 탭 인라인 편집은 이 엔드포인트에 benefits 부분 PATCH.
- **단일 진실원천 유지**: 셀과 fx 계산식은 **반드시 같은 후보**(다중 URL 시 `source_product_id` 첨부). ⭐완전한 B 규칙(§4탭) 그대로.
- 매트릭스 자동 갱신: `window.loadMatrix` 노출 경로 재사용(크롤 후 갱신과 동일).
- 로그 위젯: surf(표면)·buy(최종매입가 fx) 별도 실시간 — 혜택 변경 후 재크롤·재계산 시 반영.

### 4.3 가격 탭 레이아웃

소싱처 카드 안에 혜택 표: `혜택명 | 유형(정적/동적 배지) | 반영 방식(드롭다운) | 값(정적=입력칸/동적=읽기전용)`. 카드 하단(또는 탭 하단)에 **전파 흐름 띠**: `혜택 수정 → fx 재계산 → 셀·매트릭스·로그 → 최종매입가`. 표면노출가/혜택 **수집 방식 배지**(API/HTML/DOM)도 함께 표기(§3.3과 동일 규율).

## 5. B — 전체크롤 강건화 설계

목표: **전체 크롤이 가이드(1~6탭)에 적힌 로직을 정확히 따르고, 어긋나면 조용히 틀린 값을 저장하지 않는다.** 가이드 카드(`crawl_guide` JSON)를 **기계가 읽는 규격**으로 격상해, 크롤 결과를 그 규격으로 검증한다.

### 5.1 가이드 = 크롤 검증 게이트

- 각 소싱처 `crawl_guide`의 `stock_rules`(품절/한정 마커)·`pricing.benefits`(혜택 규격)·검증 체크리스트를 **런타임 검증 규칙**으로 사용.
- 전체 크롤 결과 저장 직전, 각 옵션 값을 게이트에 통과시킴:
  - 재고가 한정수량 표기(`qty_patterns`)를 만났는데 실수량 N을 못 뽑았으면 → **fail loud**(저장 안 함, "크롤실패" 표기), 충분(999) 둔갑 금지.
  - 표면가/혜택이 규격과 불일치(예: 음수, base 위반) → fail loud.
- 원칙: **"수집값이 가이드 로직과 불일치 시 크롤실패 처리"**(체크리스트 `integrity_fail_loud`) 를 코드로 강제.

### 5.2 알려진 누수 차단 (현행 갭 — 메모리 근거)

| 갭 | 현상 | 수정 |
|---|---|---|
| BG_PARSE 혜택 미저장 | 라이브 확장크롤(르무통·SSF·SSG·스스)이 price·stock만 저장, 혜택 누락 | `toItemBG`/`save_crawl_result`에 혜택 필드 배선([[project_ssg_benefits_never_refreshed_by_live_crawl]]) |
| per-SKU 재고 미영속 | 스스 확장이 SKU 재고 긁어도 `current_stock` 영속 안 함 → 전부 999 | `_persist_option_stocks` 경로 보장(이미 수정, 회귀 방지 테스트) |
| 일시 실패 → 소싱처 0% | 매 런 ~1개 URL 일시 타임아웃/빈그랩 → status=error 정체, stale 가격 잔존 | 재시도(무신사 goodsPrice 3회+최종재시도패스) + 동시창 축소([[project_musinsa_transient_price_loss_retry]]) |
| 하드리셋+null 미덮음 | price=null이면 last_price 안 덮어 stale 가격 정체 | finalize 단계서 유효가격 없는 옵션 crawl_blocked([[project_crawl_reset_block]]) |

### 5.3 강건화 원칙 (코드화)

1. **재크롤 시 먼저 리셋**(NULL→pending) 후 채움 — 옛값 잔존 금지.
2. **폴백가 금지** — 매칭/크롤 실패 시 평균·최저 대표가 금지, "가격없음+크롤실패" 표면화.
3. **동시·고속에도 정확** — 상품 간 값 안 섞임(per-unit 타임아웃·격리).
4. **검증 체크리스트 18항목**을 신규 소싱처·재크롤에 게이트로 적용.
