# 소싱처 단일 명부 통합 — 설계 (2026-06-30)

> **계층**: 프로젝트 spec / **상위**: [[project_url_store_fragmentation]] · O18 · O19 · 2026-06-29 소싱처추가카드 spec
> **목표**: 프로그램에 등록된 소싱처를 **하나의 명부(레지스트리)** 로 관리. 모든 화면이 그 명부에서 이름·로고를 읽는다(드리프트·중복 소멸). UI를 합치는 게 아니라 **데이터·관리 지점**을 하나로.

---

## 1. 문제 (현재)

소싱처 "명부"가 3곳에 분열:
1. **하드코딩 builtin** `SOURCES`(source_registry.py:15) — 6개
2. **`SourcingSource`** 테이블 — 커스텀(hmall·lotteimall)
3. **`SourceRegistry`** 테이블 — 사전·가이드용(name/main_url/crawl_guide), `OptionSourceUrl.source_id` FK

추가로 **이름(라벨)이 5곳에 하드코딩**: source_registry.py `SOURCES`/`get_labels`, api.py `_BUILTIN_SOURCE_LABELS`, track.py `SOURCE_LABELS`, crawl_log.js `SOURCE_LABELS`, option_url_modal.js `abbr`.

결과: 같은 소싱처가 화면마다 다른 이름/중복으로 뜸(예: "현대H몰"(운영) vs "현대홈쇼핑(hmall)"(사전) 빈 쌍둥이). 이름 변경 불가. 로고 일관성 없음.

**진단(조사 완료)**: `source_key`(hmall·musinsa…)가 이미 95% 표면의 공통 식별자. SourceRegistry.id 는 레거시 OptionSourceUrl 한정.

## 2. 목표 / 비목표

**목표**
- 소싱처 명부 = **단일 원천**(source_key 기준). 추가/이름변경/로고/삭제 = **한 곳(소싱처 사전)**.
- 모든 화면(사전·크롤링가이드·크롤위젯·매트릭스보기·셀·옵션매트릭스)이 그 명부에서 **이름·로고를 읽음**.
- **이름(label) = 수정 가능한 껍데기** (source_key 고정, 크롤 안 깨짐).
- **로고 = URL 입력 → favicon 자동** (기존 `/api/sources/probe` + `brand_icon`).
- 양방향(한 테이블이라 본질적으로 동기화).

**비목표 (YAGNI / 안전선)**
- 화면 레이아웃 합치기 ❌ (각 화면 그대로, 읽기만 통일).
- 상세 URL 공유 ❌ (sample URL·크롤 URL 은 명부 밖, 각자 유지).
- `OptionSourceUrl` FK 재키잉 ❌ / 크롤 배관(`BundleSourceUrl.source_key`·`SourceProduct.site`) ❌ — **무결성 위험이라 안 건드림**.
- `SourceRegistry` 테이블 물리 삭제 ❌ (OptionSourceUrl FK 보존 위해 잔존, 단 사전 UI는 더 이상 이걸 안 봄).

## 3. 설계

### 3.1 단일 명부 = `SourcingSource` 확장
모든 소싱처(빌트인 6 + 커스텀)를 `SourcingSource` 한 테이블에.
- 신규 컬럼: `is_builtin`(Boolean), `crawl_guide`(Text, JSON — 기존 SourceRegistry.crawl_guide 이관).
- 기존 컬럼 활용: `source_key`(고정·UNIQUE), `label`(수정), `domain`, `favicon_url`/`logo_color`/`logo_letter`(로고), `is_active`(숨김 토글), `has_adapter`, `sort_order`.
- **빌트인 6 seed**: source_key=lemouton/musinsa/ssf/ssg/lotteon/ss_lemouton, is_builtin=True. 이미 있으면 skip(idempotent).

### 3.2 이름·로고 단일 원천화
- `source_registry.get_labels()` → 명부(get_all_sources)에서 동적 생성. 하드코딩 dict 제거.
- 파이썬 소비처(api.py `_BUILTIN_SOURCE_LABELS`, track.py `SOURCE_LABELS`) → `get_labels()` 호출로 교체.
- JS(crawl_log.js `SOURCE_LABELS`, option_url_modal.js) → 서버가 `window.DATA.sourceLabels`(또는 `/api/sources/roster`)로 주입 → 하드코딩 제거. 미주입 시 source_key 폴백(안전).

### 3.3 소싱처 사전 = 명부 관리 지점
- `/source-registry` 페이지·API를 **명부(SourcingSource)** 기준으로 전환(기존 SourceRegistry 대신).
- 기능: 목록(get_all_sources) · 이름변경(label) · 로고(URL→probe→favicon) · 추가(probe→SourcingSource) · 숨김/표시(is_active) · 삭제(커스텀만; 빌트인은 숨김만, 삭제 금지 가드).
- 로고 렌더 = 기존 `brand_icon(label, domain/favicon_url)` 재사용.

### 3.4 크롤링 가이드 = 명부 뷰
- overview/add/map 이 명부(source_key) 순회 + `crawl_guide`(SourcingSource.crawl_guide)를 읽음.
- 기존 sourcing_guide.py 의 SourceRegistry 의존을 명부로 전환. 가이드 카드(6항목·sample_urls·stage_progress·update_requested)는 SourcingSource.crawl_guide JSON 에 그대로.
- 2026-06-29 존재검사 게이트/단계진행/병합은 명부 기준으로 재배선(중복 자체가 구조적으로 사라지므로 게이트는 방어로 잔존).

### 3.5 매트릭스/셀/옵션매트릭스 = 명부 라벨 사용
- `_option_matrix_data` 가 라벨을 `get_labels()`(명부)에서 얻도록 통일(이미 일부 사용). `_key_to_regid` 도메인 매칭은 **표시 라벨 한정으로 유지**(OptionSourceUrl 경로 보존). 빈 트윈 dedup(O19)은 명부 단일화로 사실상 불필요해지나 방어로 잔존.
- 크롤 위젯: `window.DATA.sourceLabels` 로 명부 라벨 주입.

### 3.6 운영센터 탭 숨김
- 사이드바에서 `/sources`(운영센터) 항목 숨김(코드·데이터 보존, 가역). 직접 URL 접근은 유지.

### 3.7 판매처(요구 3) — 별도 작은 추가
- 판매처(마켓) **이름 수정(껍데기)**만 가능하게. 소싱처 통합과 분리, 통합 완료 후 처리. (본 spec 범위 밖, 후속 플랜.)

## 4. 마이그레이션 (안전·점진·가역)

1. **컬럼 추가** (SourcingSource.is_builtin·crawl_guide) — 기존 데이터 무손상(Alembic + ensure-columns 양 dialect).
2. **빌트인 seed** (idempotent) — source_key 일치, 이름은 기존 라벨로 초기화.
3. **crawl_guide 이관** — 기존 SourceRegistry.crawl_guide 를 도메인/이름 → source_key 매칭으로 SourcingSource.crawl_guide 복사. **백업(JSON 덤프) 후**, 매칭 실패분 로그.
4. **표면 전환 1개씩** (사전 → 가이드 → 라벨 소비처 → 위젯/매트릭스 라벨), 각 단계 라이브 검증.
5. **운영센터 숨김**.
6. 각 단계 독립 커밋 → 롤백 가능. OptionSourceUrl·크롤 배관 불변.

## 5. 데이터 무결성 원칙

- source_key 불변 (크롤 키). label 만 수정.
- 빌트인 삭제 금지(숨김만). 커스텀 삭제는 OptionSourceUrl 참조 0 확인(기존 가드 재사용).
- 이관은 복사(원본 SourceRegistry 보존) → 검증 후에도 원본 유지(롤백 안전망).

## 6. 테스트

- 단위: 빌트인 seed idempotent · get_labels 명부 동적 생성 · label 수정 후 source_key 불변 · crawl_guide 이관 매칭(도메인→key) · 빌트인 삭제 차단.
- 통합: 사전에서 이름 변경 → 가이드·위젯·매트릭스 라벨 반영(라이브). 중복 소싱처 추가 불가(게이트). 운영센터 사이드바 미표시.
- 회귀: 크롤 동작(source_key 경로)·OptionSourceUrl 매핑 무변경. sourcing 테스트 전건 PASS.
