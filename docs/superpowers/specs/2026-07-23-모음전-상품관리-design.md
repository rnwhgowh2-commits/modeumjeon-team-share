## 1. 무엇을 만드나

모음전 모드에 **「상품관리」** 메뉴 하나를 새로 만든다. 여기서 사장님은

1. **전체 현황을 본다** — 마켓 > 계정 > 판매중/품절/판매중지/승인대기 건수
2. **모음전 상품을 고른다** — 28만 건에서 이름·브랜드·상품번호로 찾아 즐겨찾기
3. **고른 상품을 본다·고친다** — 마켓별 상품명·판매가·배송·옵션·이미지

### 왜 필요한가

지금은 **마켓에 뭐가 올라가 있는지 우리 프로그램이 모른다.** 아는 것은 두 가지뿐이다.

| 아는 것 | 어디에 | 한계 |
|---|---|---|
| 우리가 직접 연동한 모음전 구성 | `set_channels` | 사장님이 손으로 이은 것만 |
| 우리가 대량등록으로 올린 상품 | `product_drafts` | 프로그램으로 올린 것만 |

더망고도 이 한계는 같다. 더망고 원장은 **소싱처에서 수집한 상품**이고(화면의 「상품수집사이트」·「마켓등록/미등록」 필터가 근거), 마켓 등록 여부는 「내가 보냈다」는 자기 기록으로만 안다. **더망고를 쓰기 전에 올린 상품, 마켓에서 직접 올린 상품은 영영 안 보인다.**

이 설계는 **마켓에서 거꾸로 읽어와** 그 빈칸을 메운다.

## 3. 이 조사에서 고친 것 — 옥션·G마켓 조용한 실패

상품관리와 별개로, 지도 정독 중 **살아있는 버그**를 찾아 먼저 고쳤다.

ESM `POST /item/v1/goods/search` 는 검색·상태 조건을 **`query` 객체 안**에 넣어야 한다. 최상위에 두면 **에러 없이 조건을 통째로 무시**하고 전체를 돌려준다.

| 요청 모양 | 「판매중만」 결과 |
|---|---|
| 최상위 `sellStatus=11` (고치기 전) | 3,260 = 전체 (무시) |
| `query.sellStatus=[11]` (지도 example) | **3,150** |

같은 방식으로 판매중지 109 · 품절 0 · `keyword="니트"` 46 · 없는 낱말 0 · 옥션만 1,605 · 지마켓만 1,655.

**낸 손해 2건**
- `registration/send_more.py:70` — 옥션 상품 등록 시 「판매중 기존 상품」에서 출하지·발송정책을 베끼는데, 필터가 무시되어 **판매중지 상품을 본보기로 집을 수 있었다.**
- `webapp/routes/live_send_test.py` — 「ESM keyword 는 무시됨」이라 결론짓고 5,000건을 10페이지 훑어 22초 걸려 거르는 우회로가 들어가 있었다. **원인은 우리 요청 모양이었다.**

수정: `shared/platforms/esm/products.py::search_goods` + 회귀 테스트 `tests/platforms/test_esm_goods_search_query.py` 9건.

## 4. 데이터 모델

### 4-1. 새 테이블 2개

**`market_products` — 마켓 상품 머리글 캐시** (약 28만 행)

| 열 | 뜻 | 비고 |
|---|---|---|
| `market`, `account_key` | 어느 마켓 어느 계정 | |
| `market_product_id` | 마켓 상품번호 | 스스=channelProductNo · 롯데온=spdNo · ESM=goodsNo |
| `site_product_id` | 사이트별 번호 | ESM 전용(옥션/지마켓이 갈림) |
| `name` | 상품명 | ★ 롯데온은 `&lt;` HTML 이스케이프로 온다 — **풀어서 저장** |
| `status` | 통일 4상태 | `sale` / `soldout` / `stopped` / `waiting` |
| `raw_status` | 마켓 원본 코드 | 거짓 통일 방지 — 원본을 반드시 남긴다 |
| `sale_price` | 판매가 | 마켓이 안 주면 `NULL`(0 금지) |
| `registered_at` | 마켓 등록일 | |
| `synced_at` | 마지막으로 확인한 시각 | 화면에 그대로 표시 |
| `set_id` | 모음전 상품으로 묶였으면 그 번호 | `NULL` = 아직 안 담음 |

- 유일 제약 `(market, account_key, market_product_id)`
- 인덱스 `(market, account_key, status)` · `(set_id)` · 상품명 부분검색용 GIN(pg_trgm)
- **지운 행은 지우지 않는다** — `deleted_at` 으로 표시(마켓에서 사라진 것도 이력)

**`market_product_counts` — 대시보드 건수 스냅샷** (수십 행)

`market` · `account_key` · `status` · `count` · `measured_at` · `source`(`api` \| `cache`)
→ 화면은 이 표만 읽는다. **대시보드는 항상 즉시 뜬다.**

### 4-2. 기존 것을 쓴다 — 즐겨찾기·옵션 연동

「1개 상품 → 여러 마켓 상품」 구조는 **이미 있다.** 새로 만들지 않는다.

```
product_sets          내가 관리하는 상품 1건 (= 모음전 상품, 즐겨찾기의 실체)
  └ set_channels      마켓 × 계정마다 1행 — market_product_id      ← 1:N 이 여기서 갈림
      └ set_channel_options   옵션마다 1행 — market_option_id + 마켓 재고·가격
```

사장님이 말씀하신 **「모음전 상세페이지의 옵션 ↔ 마켓 상품의 옵션 연동」이 `set_channel_options`** 다. 매칭 상태(`matched`/`unmatched`/`ambiguous`/`duplicate`)까지 이미 갖고 있다.

**「모음전으로 담기」가 하는 일**
1. `product_sets` 에 1건 생성 (없으면)
2. 고른 마켓 상품마다 `set_channels` 1행
3. 그 상품의 옵션을 마켓에서 받아와 `set_channel_options` 채우기
4. `market_products.set_id` 기록

## 5. 동기화

### 5-1. 밤에 한 번 전체 훑기

새벽 3시. 마켓별 페이징으로 머리글만 받아 `market_products` 에 upsert.

| 마켓 | 페이지당 | 예상 호출 |
|---|---:|---:|
| 롯데온 | 100 | 1,402 |
| 11번가 | 100 | 910 |
| 쿠팡 | 100 | 292 |
| 스마트스토어 | 100 | 96 |
| 옥션·G마켓 | 500 | 20 |
| | | **약 2,720회** |

실측 속도(11번가 131콜 77초 = 0.59초/콜) 기준 **30~60분** 예상. 밤에 도는 일이라 문제 없다.

**계정 하나가 실패해도 나머지는 계속한다.** 실패한 계정은 화면에 「마지막 확인: 어제」로 정직하게 뜬다 — 조용히 옛 숫자를 최신인 척 보여주지 않는다.

### 5-2. 낮에는

- 계정 카드마다 **「지금 동기화」** 버튼 → 그 계정만 다시 훑는다
- 대시보드 건수는 **즉답 되는 마켓만 그때그때 새로** 받는다(스스·롯데온·ESM, 몇 초)
- 쿠팡은 `inflow-status` 로 총건수만 즉답(상태별은 캐시 기준)

### 5-3. 나중에 — 바뀐 것만 받기

지도 정독으로 확인한 델타 조회 수단(이번 범위 아님, 기록만):
- 11번가 `schDateType=4`(상품수정일) + 기간
- ESM `query.updatedDate.from/to`
- 쿠팡 `time-frame`(생성일만 — 수정은 못 잡음)
- 스마트스토어 `periodType`+`fromDate`/`toDate` (실측: 먹힘)
- 롯데온 — 등록일만, 수정일 없음

## 6. 화면

모음전 모드 사이드바 `판매` 그룹에 **「상품관리」** 한 줄 추가. 안에서 가로탭.

### 6-1. 대시보드

- 마켓 > 계정 > 4상태 건수. 「마지막 확인: 오늘 새벽 3시」 항상 표시
- 탭 2개: **모음전**(기본) / 대량등록
  - **모음전 탭** = `set_id` 가 있는 상품 (사장님이 담은 것)
  - **대량등록 탭** = `product_drafts` 로 우리가 올린 것
  - 대량등록 모드의 기존 「📦 상품관리」 탭은 **이 화면으로 링크만** 남긴다(중복 제거)
- 성과 숫자(매출·마진)는 기존 주문내역·마진계산기에서 끌어온다

### 6-2. 모음전 상품 담기

- 검색: 상품명 · 브랜드 · 상품번호 (전부 우리 캐시에서 → **즉시**)
- 결과에 마켓·계정 뱃지. 여러 개 골라 **한 상품으로 묶어** 담기
- 같은 이름이 다른 마켓에도 있으면 **「이것들도 같은 상품인가요?」** 자동 제안
  (자동 확정 금지 — 확인은 사장님이)

### 6-3. 상품 상세·편집

- 조회: 마켓별 상품명·판매가·배송정책·이미지 / 옵션 연동 상태
- 바로가기 **2개**: 모음전 구성 화면 + 손님이 보는 마켓 상품 페이지
- 편집: 상품명·카테고리·판매가·옵션·이미지·상세설명·배송·고시정보·브랜드·원산지·KC인증·금지어·태그
- **저장 = 우리 DB에만.** 「마켓에 반영」은 별도 버튼 + 변경 전/후를 보여준 뒤 전송

## 7. 함정 (실측으로 확인한 것)

1. **ESM 조건은 `query` 안에** — 밖에 두면 조용히 무시(§3)
2. **롯데온 `pageNo`·`rowsPerPage` 필수** — 빠지면 `returnCode 9000`("처리 중 오류"). 권한 문제로 오해하기 쉽다
3. **롯데온 상품명은 HTML 이스케이프**(`&lt;매장정품&gt;`) — 풀어서 저장하지 않으면 검색이 안 걸린다
4. **스마트스토어는 상품명 검색이 안 된다** — 후보 16종 전부 무시 또는 400. 캐시에서 찾는다
5. **쿠팡·11번가는 총건수 필드가 없다** — 세려면 끝까지 넘겨야 한다
6. **옥션·G마켓 마스터번호는 공용** — 사이트별 수는 `query.siteId` 로 갈라야 나온다
7. **지도 응답 필드 목록은 축약본일 수 있다** — 롯데온이 상품명을 안 주는 줄 알았으나 실제로는 25개 필드가 온다. 지도만 믿지 말고 한 번 열어볼 것
