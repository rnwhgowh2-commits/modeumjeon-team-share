# 사이드바 기능별 탭 재구성 (1단계) — 설계 문서

> 작성일: 2026-06-06
> 상태: 승인됨 (사용자 확인 완료)
> 성격: 순수 네비게이션 재배치. 페이지·라우트·active_key·데이터 변경 없음.

---

## 1. 배경 / 목적

기존 사이드바는 4개 그룹(소싱·발견 / 모음전 등록 / 판매·운영 / 정산·관리)으로,
업무 흐름보다 항목이 흩어져 있어 찾기 어렵다. **기능 위주로 6개 대분류**로 재정리한다.

이번 작업은 **단계 분할**의 1단계 = "사이드바 네비게이션 재배치"만 수행한다.
스크린샷 매핑표에 표시된 "통합"(여러 페이지를 하나로 합치기, 예: 모음전 상품관리 등록전/등록상태/신규등록)은
**이후 각각 별도 사이클(spec → plan → 구현)** 로 진행한다. 본 스펙 범위 밖이다.

## 2. 범위

### 포함
- 사이드바 그룹을 6개 대분류로 재구성하고 기존 항목을 새 그룹으로 이동
- 3개 항목을 사이드바에서 숨김(네비게이션에서만 제거)
- 크롤링 가이드 자동 주입 위치를 새 "크롤링&업로드" 그룹으로 변경

### 제외 (명시적 비범위)
- 페이지/라우트/active_key/DB/데이터 변경 — **일절 없음**
- 페이지 통합(여러 화면 합치기) — 이후 별도 사이클
- 숨긴 3개 기능의 코드/데이터 제거 — **하지 않음** (기능·URL 모두 살아있음)

## 3. 핵심 결정 사항 (Q&A 확정)

1. **작업 범위**: 통합 단위로 단계 분할 → 1단계는 사이드바 재배치만.
2. **④ 그룹 이름**: "구매" (스크린샷의 "매입" 대신).
3. **삭제 3항목 처리**: **세 개 모두 사이드바에서만 숨김, 완전 제거 없음.**
   - 의존성 조사 결과, "중복·불필요"는 마켓 업로드 설정에만 해당했고
     소싱처 계정·사입 재고는 크롤·마진의 뼈대라 제거 시 손실이 컸기 때문.
     화면 정리 목적은 "숨김"만으로 달성되므로 가장 안전한 숨김으로 통일.

### 삭제 3항목 의존성 조사 요약 (근거)
- **마켓 업로드 설정** (`/market-upload-config`): 저장 데이터 0(전부 하드코딩 더미),
  업로드 파이프라인이 읽지 않음(실제 업로드는 MarketRegistry + PriceTemplate). → 제거해도 무해했음.
- **소싱처 계정** (`/accounts/sourcing`): 크롤러 자동 로그인이 이 ID/PW/세션을 사용
  ([api_pricing.py 자동로그인], [login_wizard.py]). 제거 시 가격 자동 수집 중단·계정 데이터 소실. 중복 아님.
- **사입 재고 연동** (`/boxhero`): 평균 매입가·재고·마진정책이 원가(COGS)·마진계산·업로드가·출고검증으로 연결
  ([cogs.py], [boxhero_margin.py], [unified.py]). 제거 시 마진/원가 붕괴 + 영구 데이터 손실. 중복 아님.
- 결론: 셋 다 "숨김"으로 통일(완전 제거 없음) → 이번 작업이 크롤·마진·업로드를 일절 건드리지 않음.

## 4. 신구조 (목표 레이아웃)

### 최상단 standalone (유지)
- 🏠 홈 (`/`, active_key=home)
- 🗺 로드맵 (`/roadmap`, active_key=roadmap) — 코드 주입 유지

### 6개 대분류 (순서·이모지·색상은 제안값; 편집 메뉴로 변경 가능)

| # | 그룹 | 이모지 | 색상(제안) | 항목 (active_key) |
|---|------|--------|-----------|-------------------|
| ① | 모음전 상품관리 | 📦 | #3182F6 | 신규 모음전 등록(bundles_new) · 모음전 상품관리(bundles) · 기존 마켓 연동(bundles_migrate) |
| ② | 매핑 현황 | 🔗 | #FF9500 | 소싱처 운영센터(sources) · 미맵핑 큐(queue) · 맵핑(mapping) |
| ③ | 크롤링&업로드 | 🛒 | #03C75A | 크롤링 가이드(sourcing_guide) · 소싱처 사전(source_registry) · 업로드 실패함(dlq) · 판매처 계정(accounts_upload) |
| ④ | 구매 | 🛍 | #EF4444 | 가격·재고 추적(track) |
| ⑤ | 판매 | 💰 | #A855F7 | 템플릿(templates) · 주문 내역(orders_list) · 매출 관리(orders_sales) · 마진 계산기(orders_margin) |
| ⑥ | 기타 | ⚙️ | #6B7280 | 휴지통·변경 이력(trash) · 알림 채널 설정(alerts) |

### 사이드바에서 숨김 (기능·URL·데이터 유지)
- 소싱처 계정 (`/accounts/sourcing`, accounts_sourcing)
- 마켓 업로드 설정 (`/market-upload-config`, market_upload)
- 사입 재고 연동 (`/boxhero`, boxhero)

### 배지 유지
- 미맵핑 큐: badge_key=unmapped
- 업로드 실패함: badge_key=failed

## 5. 구현 (고칠 파일 2곳)

### 5.1 `data/sidebar_layout.json` — 라이브 레이아웃 재작성
- 현재 4 stages → 위 6 stages 로 교체. 각 항목의 `id`/`url`/`active_key`/`badge_key`는 **기존 값 그대로** 유지(이동만).
- 크롤링 가이드 항목(`i_crawl_guide`)을 ③ 그룹에 명시적으로 포함(주입 의존 제거).
- 숨김 3항목은 stages/standalone 어디에도 넣지 않음.
- 런타임은 mtime 캐시(`_load`)라 파일 저장 시 자동 반영.

### 5.2 `webapp/routes/api_sidebar.py` — `_default_layout()` 동기화
- `_default_layout()` 을 5.1과 **동일한 6그룹 구조**로 갱신(리셋·최초부팅 일치).
- 크롤링 가이드 주입 로직(`get_layout_for_template`): 현재 `s1` 대상 → 새 크롤링&업로드 그룹 id 대상으로 변경.
  레이아웃에 이미 포함되면 `_has_item_id` 가 True 라 주입은 no-op 이 되지만, 방어적으로 대상 id를 맞춰둔다.
- 로드맵 주입(standalone)은 그대로 유지.

### 영향 없음 확인
- `partials/sidebar.html` 은 `sidebar_layout`(stages/items)만 순회 렌더 → 구조 변경에 자동 대응. 수정 불필요.
- 하이라이트는 각 페이지의 `active` 값 ↔ 항목 `active_key` 매칭 → active_key 불변이므로 유지됨.
- stage `id` 는 드롭존/data-id 용도뿐이라 재배치에 영향 없음.

## 6. 검증 (live-browser-verify)

- [ ] 사이드바에 6개 그룹이 위 순서로 표시됨
- [ ] 각 그룹의 항목이 표대로 위치함 (22개 항목 누락·중복 없음)
- [ ] 숨김 3개(소싱처 계정·마켓 업로드 설정·사입 재고 연동)가 사이드바에 없음
- [ ] 숨긴 3개의 URL 직접 접속 시 페이지 정상(기능 살아있음): `/accounts/sourcing`, `/market-upload-config`, `/boxhero`
- [ ] 각 항목 클릭 → 해당 페이지 이동 + 사이드바 하이라이트 정상
- [ ] 배지(미맵핑 큐 unmapped, 업로드 실패함 failed) 정상 표시
- [ ] 크롤링 가이드(`/sourcing-guide/`)가 ③ 그룹에 1번만 표시(중복 주입 없음)
- [ ] 리셋(`POST /api/sidebar/layout/reset`) 후에도 6그룹 구조 유지

## 7. 워크트리 / 배포

- 현재 브랜치 `feature/crawl-worker-queue`. 작업 후 master 머지 + 라이브 서버 반영까지 완료해야 종료(글로벌 6원칙).

## 8. 후속(별도 사이클, 본 스펙 밖)

- 모음전 상품관리 통합: 신규 등록 + 상품관리 + 기존 마켓 연동 → 등록전/등록상태/신규등록 하위 상태로 통합
- 매핑 현황 통합: 소싱처 운영센터 + 미맵핑 큐 + 맵핑 → URL(정상/에러) + 재고 매핑으로 통합
