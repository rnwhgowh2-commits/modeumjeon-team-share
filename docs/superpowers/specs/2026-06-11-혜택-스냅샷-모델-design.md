# 혜택 = 소싱처 기본셋팅 → 모음전 스냅샷 복사 모델 — 설계

> 2026-06-11. 소싱처 혜택을 "기본 셋팅"으로, 모음전은 생성 시점 값을 **복사(스냅샷)**해 이후 독립.
> 관계: [[2026-06-07-최종매입가-계산엔진-design.md]](compute_final_price 엔진), [[project_fx_benefit_model]], [[project_url_store_fragmentation]].
> 사용자 확정(2026-06-11): A3-1(슬림 배너) · B1(체크박스 2중잠금) · 기존 모음전=현재 기본값으로 통일.
> 상태: **구현 진행** (Phase 1 엔진/마이그레이션부터).

---

## 1. 목표 모델 (사용자 확정)

1. **크롤가이드 = 소싱처별 기본 셋팅(원천)** — `SourceBenefitTemplate`.
2. **모음전 신규 생성 = 그 시점 기본값을 모음전 안으로 복사(스냅샷)**. 이후 **독립**.
3. 소싱처 기본값이 바뀌어도 **기존 모음전 불변** (신규 혜택 추가 포함 — 영향 0).
4. 모음전마다 개별 수정.
5. **A3-1 초기화**: 이 모음전을 **현재** 소싱처 기본값으로 다시 복사(되돌리기).
6. **B1 따라쓰기**: 소싱처 기본값을 **모든 모음전**에 덮어쓰기 — 2중 잠금(체크박스).

## 2. 저장 구조 (엔진 변경 최소)

기존 테이블 재사용:
- `SourceBenefitTemplate` (source_id) = **기본 셋팅 카탈로그**. 엔진이 직접 쓰지 않고 *시드/초기화/따라쓰기*의 원본으로만 사용.
- `OptionBenefitOverride` (canonical_sku, source_id) = **모음전 옵션의 자기 복사본(스냅샷)**. `template_id=NULL`(standalone)로 저장. 모음전 혜택의 실제 값 = 여기.
- `SourceProduct.dynamic_benefits_json` = 크롤 동적값(무신사머니·등급적립 등) — 스냅샷 대상 아님, 계속 실시간.

### 핵심 엔진 게이트 (compute_breakdown)
"이 옵션이 스냅샷되었는가" = **그 (sku, source_id)에 override 행이 하나라도 있으면 스냅샷됨**.
```
if ovr_items:   # 스냅샷 모드 → 소싱처 템플릿 무시(독립). override 가 완전한 집합.
    effective = [('ovr_new', o) for o in ovr_items]
else:           # 미스냅샷(하위호환) → 기존처럼 소싱처 템플릿 사용
    effective = [('tpl', t) for t in tpl_items]
# 이후 dynamic_benefits 주입·카드 로직은 동일
```
- 게이트는 **마이그레이션과 동시에** 적용해야 안전(부분 override 옵션의 가격 왜곡 방지). 둘은 원자적.
- 미스냅샷 옵션(override 0건)은 **기존 동작 byte-identical** → 회귀 안전.

## 3. 마이그레이션 (1회, 기존 모음전 → 통일)

각 (모음전 옵션 canonical_sku, source_id) 에 대해:
1. 기존 override 전부 삭제(통일 정책 — 개별 커스터마이즈 폐기, 사용자 확정).
2. 그 source 의 `SourceBenefitTemplate` 항목들을 **standalone override 로 복제**(value/type/enabled/category/apply_mode/pay_method/channel/sort_order 복사, template_id=NULL).
- 결과: 모든 모음전 옵션이 스냅샷 모드. 가격 = 현재 소싱처 기본값.
- **idempotent**: 재실행 시 동일 결과(삭제 후 재복제).
- 대상 옵션 = `_options_by_bundle_code` 로 전 모음전 순회 × 그 옵션이 가진 소싱처 URL 의 source_id 들.
- 백업 우선: 실행 전 override 테이블 덤프.

## 4. 신규 생성 훅 (스냅샷 복사)

모음전 생성(`/bundles/new` POST) 또는 옵션 생성 시:
- 그 옵션이 연결된 각 source_id 에 대해, override 없으면 §3-2 와 동일하게 템플릿을 standalone override 로 복제.
- 헬퍼 `snapshot_bundle_from_templates(session, bundle_code, source_ids=None)` 재사용(마이그레이션·초기화·생성 공용).

## 5. A3-1 — 초기화 + "기본값과 다름" 배너

- **판정 API** `GET /api/source-benefits/diff/<bundle_code>/<source_id>` → `{differs:bool, items:[{name, current, default}]}`.
  - 모음전 override 값 ↔ 현재 SourceBenefitTemplate 값 비교(이름 매칭). 다른 항목 목록.
- **배너**: fx 팝업 상단, differs 일 때만 슬림 앰버 바 + [차이][기본값으로 초기화].
- **초기화** = `POST /overrides/reset-bundle` 의미 변경: (기존)삭제 → (신규)**현재 템플릿을 standalone override 로 재복제**(snapshot_bundle_from_templates). 삭제하면 미스냅샷→템플릿 폴백이라 "현재 기본값"과 결과는 같지만, 일관성 위해 재복제 방식.

## 6. B1 — "모든 모음전에 따라쓰기" (2중 잠금)

- **엔드포인트** `POST /api/source-benefits/templates/<source_id>/apply-to-all` payload `{confirm:true}`.
  - 전 모음전 옵션(해당 source) override 를 현재 템플릿으로 재복제(snapshot). 영향 N 반환.
- **UI**: 크롤가이드 ③ 카드 하단 위험 카드. 체크박스 ON → 버튼 활성 → 실행. 영향 모음전 수 표기.

## 7. 이전 세션 작업 정리

- 유지: `set-value-bundle`(✎ 모음전 전체 적용) — 스냅샷 모델과 호환(standalone override upsert).
- 변경: `reset-bundle` 의미 = 삭제 → 현재 템플릿 재복제.
- 제거: fx 행별 ↺ 버튼(A2 안) → A3-1 배너로 대체.

## 8. 무결성 / 회귀 / 리스크

- **blast radius**: compute_breakdown 게이트 = 전 매트릭스 가격. 게이트+마이그레이션 동시 적용.
- **회귀 테스트(필수)**: 마이그레이션 전후, **override 없던 옵션**의 compute_breakdown final_price 동일(트랜잭션 롤백 비교). override 있던 옵션은 통일로 변경(허용).
- 음수가드·동적주입·카드토글 로직 불변.
- 크롤 동적값은 스냅샷 무관(계속 실시간).

## 9. 마일스톤

- **M1**: 엔진 게이트 + snapshot 헬퍼 + 마이그레이션 스크립트 + 회귀 테스트(롤백 비교). DB 적용.
- **M2**: 생성 훅 + reset-bundle 재복제 + diff API.
- **M3**: A3-1 배너(fx 팝업) + 행 ↺ 제거.
- **M4**: B1 따라쓰기 엔드포인트 + 크롤가이드 ③ 2중잠금 UI.
- 각 단계 라이브 검증 + 7섹션 보고.
