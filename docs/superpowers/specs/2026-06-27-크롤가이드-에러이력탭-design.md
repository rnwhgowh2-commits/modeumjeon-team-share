## 1. 목적·배경

전체크롤 운영 중 **재고 오류·가격 오류**가 반복 발생했다. 매번 같은 원인(저장 배선 누락·정규식 미스·중복행 폴백·혜택 미저장 등)이 다른 소싱처에서 재현된다.

**목적** — 신규 소싱처 추가·크롤 방식 업데이트 전에 **먼저 보고 같은 실수를 막는** "실제 발생 에러 카탈로그"를 데이터 지도(`/sourcing-guide/map`)에 둔다. 보기·원문 둘 다 대칭 반영.

**왜 기존 §5 재발로 부족한가** — 현재 §5/탭6 '재발'은 "증상 → 첫 확인 위치" 6줄 빠른 인덱스다. (가) 실제 발생 이력 전체를 담지 못하고 (나) **백엔드/프론트엔드/기타 원인 분류**가 없어 "신규 작업 시 어느 층을 점검할지" 신호가 약하다. 새 카탈로그가 이를 **상위호환**으로 흡수한다.

## 2. 범위 (YAGNI)

**포함**
- 보기(map.html): 기존 탭6 `🔁 재발` → **`⚠️ 에러이력`(s6)** 으로 교체. 시안 B(카드 + 좌측 그룹 컬러바).
- 원문(.md): 기존 `§5. 자주 재발하는 문제` → **`§5. 에러 이력 — 신규 추가 전 필독`** 으로 교체(동일 카탈로그).
- 에러 25건 전수 수록(재고 11 · 가격 9 · 기타 5). 기존 재발 6항목은 해당 카탈로그 카드로 전부 흡수.
- 교차 참조(`→ §5 재발` / `→ 6 재발`) 문구를 `에러이력`으로 갱신.

**불포함(현 작업 아님)**
- 인터랙티브 필터(시안 C)·매트릭스 보드(시안 D) — 채택 안 함(B 카드).
- 에러를 DB/JSON 데이터로 빼서 동적 렌더 — 정적 카드로 충분(가이드는 손으로 갱신).
- 에러 자동 수집·링크(메모리 `[[...]]` 자동화) — 후속 가능, 지금은 수기.

## 3. 분류 2축 (확정)

- **그룹**(무엇이 틀렸나): `📦 재고` · `💰 가격` · `🛠️ 기타`
- **코드 레이어**(어디 고쳐야 하나): `백엔드(BE)` · `프론트(FE, 확장·매트릭스 JS 포함)` · `기타원인(인프라·배포·데이터·운영)`
- 한 에러가 두 층이면 주(主) 배지 + 보조 배지(예: 비활성옵션 가격누출 = BE 主 + FE).

> FE = 사용자 화면 JS **및 크롬 확장 추출기**(확장도 프론트 코드로 본다). BE = Flask·크롤러·저장·계산. 기타 = docker 배포·DB stale·버전 분열·honesty 등 코드 외/구조.

## 5. UI 설계 — 보기(map.html)

- 탭 스트립 6번 `🔁 재발`(빨강 newt 클래스) → `⚠️ 에러이력` 으로 라벨·내용 교체. `data-s="s6"`·키보드 6 유지(탭 순서·개수 불변 → JS 변경 불필요).
- 섹션 `#s6` 내부: 리드 한 줄 + 빠른복귀 안내(2줄) + 그룹 3블록(재고/가격/기타), 각 블록은 **2열 카드 그리드**.
- 카드 마크업(시안 B): 좌측 6px 그룹 컬러바(재고=파랑·가격=초록·기타=회색) + 본문(상단 그룹배지+레이어배지+소싱처 / 증상(굵게) / 원인 / 재발방지(초록) / 코드(mono)).
- 색·폰트·카드 톤은 기존 `.sgmap` 토큰(`--ink/--blue/--green/--red/--sub`, Pretendard) 재사용 — 신규 CSS는 `.sgmap` 스코프 안에 카드/배지 클래스만 추가.
- 시안 HTML(`바탕화면/모음전 에러이력 탭 시안 v1.html`)의 B 렌더가 마크업 레퍼런스.
