# 소싱처 사전 → 크롤링 가이드 통합 (중복 화면 제거) — 설계 (2026-06-30)

> **상위**: 2026-06-30 소싱처-단일명부-통합. 단일 명부 통합 이후 사전·가이드 전체보기가
> 같은 명부를 나열 → 중복. **가이드 전체보기 한 곳**으로 합치고 사전은 완전 제거.

## 1. 문제
단일 명부 통합 후 두 화면이 같은 소싱처 목록을 보여줌:
- 크롤링 가이드 > 전체보기 = 문서 보기(구조·매입가·혜택) + 행 클릭 상세
- 소싱처 사전(/source-registry) = 관리(이름·로고·숨김·삭제·정렬)
→ 비슷한 화면 2개 = 헷갈림.

## 2. 결정(사용자 확정)
1. **사전 완전 삭제** (메뉴 숨김 아님 — 코드까지)
2. **편집 = 전체보기 표 행 인라인** (✎이름·🖼로고·👁숨김·🗑삭제)
3. **편집 권한 = 로그인한 누구나** (admin 게이트 없음)
4. **추가 = 기존 2번째 카드(소싱처 추가·업데이트)** 그대로

## 3. 설계
### 3.1 사전 제거
- `webapp/templates/source_registry/list.html` 삭제
- `webapp/routes/source_registry.py` 의 페이지 라우트(`GET /source-registry`) 삭제
- **관리 API는 sourcing_guide 로 이동**(빈 블루프린트 안 남김): 이름변경·로고(URL)·숨김(is_active)·삭제·정렬. 전부 `lemouton.sourcing.roster` 서비스 호출(이미 존재).
- `webapp/routes/__init__.py` 의 source_registry 블루프린트 등록 제거
- 사이드바 `i_src_dict`(소싱처 사전) 항목 제거(default_layout + 렌더 필터)

### 3.2 가이드 전체보기 = 단일 소싱처 화면
- `overview()` 가 행마다 추가 전달: `key`(source_key)·`label`·`favicon_url`·`domain`·`is_builtin`·`is_active`·`usage`
- `overview.html`:
  - 이름 셀 = 로고(favicon, `brand_icon` 또는 img) + 이름 + `기본`/`숨김` 배지
  - 행 끝 인라인 편집(로그인 누구나): ✎이름변경 · 🖼로고URL · 👁숨김토글 · 🗑삭제(커스텀만; 빌트인은 버튼 미노출)
  - 행 클릭 = 상세 4단계 유지. **편집 버튼은 `stopPropagation`** 으로 행 클릭과 분리
  - 기존 칼럼(샘플 URL·구조 N/6·매입가·가격검증·혜택) 유지
- 신규 가이드 엔드포인트(로그인 게이트만, admin 강제 없음):
  - `PUT /sourcing-guide/api/source/<key>` {name?|logo_url?|is_active?} → roster.rename/set_logo/set_active
  - `DELETE /sourcing-guide/api/source/<key>` → roster.delete(빌트인·참조 가드)
  - `POST /sourcing-guide/api/sources/reorder` {keys[]} → sort_order
