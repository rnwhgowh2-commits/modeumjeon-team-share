## 1. 목표 (한 줄)

CS 탭 「고객문의」 구역을 채운다 — 마켓별 고객문의를 **미답변/답변완료**로 보고(인박스형), 문의형태별로 구분하며, 답변을 **작성·미리보기**까지 한다(**전송은 안전 OFF**). 더불어 **완료 건(#2 대응완료·#3 답변완료)을 7일 자동 숨김 + 수기 삭제**로 정리한다.

## 2. 첫 컷 범위 (사용자 결정: 탭 뼈대+조회 우선, 전송 OFF)

- **실조회 가능**(코드 있음): **쿠팡**(`coupang/inquiries.py` `fetch_online_inquiries`), **스마트스토어**(`smartstore/orders.py` `fetch_inquiries` — 상품문의 WAIT). 라이브 검증.
- **연동 준비 중**(문서만·미검증): 롯데온(상품QnA 99·판매자문의 179), 11번가(미확인), 스스 톡톡(별도 네이버 톡톡 API·미코딩). → 마켓 탭/형태에 "연동 준비 중" 표기(거짓 조회 금지).
- **답변 전송 = 고객 대면 발신** → 프로젝트 규율상 **`LEMOUTON_LIVE_INQUIRY_REPLY` OFF**(기본). 작성·미리보기까지만. 실전송은 마켓별 검증 후 별도.

## 3. 화면 구조 (확정 시안 3 — 인박스)

```
CS 탭 → 「고객문의」 구역
├─ 상단 가로 마켓 탭 (로고+건수 배지, #2 재사용): 전체·쿠팡·11번가·롯데온·스마트스토어
├─ 상태 탭: [미답변] [답변완료]
└─ 인박스: [좌] 문의 목록(선택)  |  [우] 선택 문의 상세 + 답변 패널
```
- 좌 목록 행: 문의형태 배지(톡톡=청록·상품문의=보라·온라인문의=파랑·판매자QnA=자홍) + 고객 + 일시 + 문의 미리보기.
- 우 상세: 형태·마켓·고객·상품·일시 + 문의 원문 + **답변 작성칸**(textarea) + 「미리보기」 + **「전송 🔒(준비 중)」**(OFF 안내).
- 답변완료 문의: 우 패널에 내 답변 표시(읽기전용) + **「삭제」**(수기 정리, §5).
- 마켓 탭 = 클라이언트 필터(항상 전 마켓 fetch → 건수 오표시 방지, #2 교훈).

## 4. 데이터 계층 — 고객문의

### 4.1 수집 서비스 (`lemouton/cs_inquiries/service.py`, 신규 모듈)

```
list_inquiries(markets, *, since, until, status='WAIT') -> {groups:{미답변,답변완료}, market_counts, warnings}
```
- 마켓별 어댑터 호출(쿠팡 `fetch_online_inquiries`·스스 `fetch_inquiries`) → 통일 dict로 정규화: `마켓·문의형태·문의ID·고객·상품·문의내용·일시·상태(미답변/답변완료)·답변내용(있으면)`.
- **미지원 마켓**(롯데온·11번가)은 조회 스킵 + `warnings`에 "연동 준비 중" 담아 표면화(#2 부분성공 패턴 재사용, 조용한 실패 금지).
- 한 마켓 실패 = warnings 담고 나머지 정상(부분 성공).

### 4.2 답변 (전송 OFF)

- `reply_inquiry_dry(market, inquiry_id, content)` — 검증·미리보기용. `LEMOUTON_LIVE_INQUIRY_REPLY` ON일 때만 실제 `reply_online_inquiry`/`reply_inquiry` 호출. 기본 OFF → 미리보기 응답만(거짓 전송 금지).
- 답변 초안 저장 = **하지 않음**(YAGNI, 첫 컷). 작성은 화면 세션 한정.

### 4.3 처리상태 저장 (`lemouton/cs_inquiries/models.py`)

`InquiryHandling` — **삭제(정리)용 최소 저장**:
| 필드 | 뜻 |
|------|-----|
| `id` PK | |
| `inquiry_key` unique | `f"{market}:{inquiry_id}"` |
| `market` | 표시용 |
| `dismissed_at` | 수기 삭제 시각(있으면 목록서 숨김) |

- 답변완료 여부는 마켓 API(answeredType) 파생. 우리가 저장하는 건 삭제 플래그뿐.

## 5. 완료건 정리 (#2·#3 공통 규칙)

**대응완료(#2 클레임)·답변완료(#3 문의) 는 완료된 지 오래되면 목록서 사라진다.**

- **자동 숨김(7일)**: 완료 항목은 **완료일 기준 최근 7일**만 노출. 7일 지나면 자동으로 목록서 빠짐(데이터 삭제 아님 — 표시 필터).
  - #2 대응완료 = `변경일`(`_change_date`, 종결 이벤트일) 기준. #3 답변완료 = 답변일(API 제공) 기준.
- **수기 삭제**: 완료/답변완료 항목의 「삭제」 → 즉시 목록서 제거(7일 전이라도). = 처리기록에 `dismissed_at` 세팅.
  - #2: `ClaimHandling.dismissed_at` 신규 컬럼(create_all 자동). #3: `InquiryHandling.dismissed_at`.
- **필터 규칙**(대응완료·답변완료 그룹): `완료일 ≥ (오늘-7일)` **그리고** `dismissed_at is None` 인 것만.
- **미답변·신규요청·대응필요는 대상 아님**(항상 노출 — 처리해야 하므로).
- **재등장 방지**: dismissed 된 항목은 다시 fetch돼도 계속 숨김(플래그 지속).

### 5.1 #2 변경 (이미 배포된 코드 수정)
- `ClaimHandling`에 `dismissed_at` 추가.
- `service.list_claims`: 대응완료 그룹을 `변경일 최근7일 ∧ dismissed_at is None`으로 필터.
- 신규 라우트 `POST /orders/cs/claims/dismiss {claim_key}` → dismissed 세팅.
- UI: 대응완료 카드에 「삭제」 버튼.

## 6. 라우트

- `GET /orders/cs/inquiries.json?markets=&status=` → {groups:{미답변,답변완료}, market_counts, warnings}.
- `POST /orders/cs/inquiries/reply-preview` `{market,inquiry_id,content}` → 미리보기(전송 OFF 시 실호출 안 함).
- `POST /orders/cs/inquiries/dismiss` `{inquiry_key,market}` → 삭제(숨김).
- (#2) `POST /orders/cs/claims/dismiss` `{claim_key}` → 대응완료 삭제.

## 7. 엣지 케이스

| 상황 | 처리 |
|------|------|
| 롯데온·11번가 문의 | 조회 스킵 + "연동 준비 중" 배너(warnings). 거짓 빈목록 아님. |
| 답변 전송 시도(OFF) | 미리보기만·"전송 준비 중" 안내. 실호출·거짓 성공 없음. |
| 완료 7일 경과 | 자동 숨김(표시 필터). 마켓 데이터·처리기록 보존. |
| 수기 삭제 후 재조회 | dismissed 지속 → 계속 숨김. |
| 삭제한 걸 되살리기 | 첫 컷 미지원(YAGNI). 필요 시 후속. |
| 한 마켓 조회 실패 | warnings 부분성공(보드 유지). |

## 8. 테스트 (TDD)

1. **list_inquiries**: 쿠팡·스스 어댑터 정규화(fixture) → 미답변/답변완료 그룹·마켓 건수. 미지원 마켓 warnings.
2. **완료 7일 필터**: 대응완료/답변완료가 `완료일 최근7일 ∧ not dismissed`만. 8일 전=숨김, dismissed=숨김, 미답변=항상.
3. **dismiss**: claims/inquiries dismiss upsert → 이후 list에서 제외.
4. **reply preview (OFF)**: LIVE OFF면 실 reply 함수 호출 안 됨(미리보기만).
5. **route**: inquiries.json 구조·dismiss·reply-preview 왕복.
6. **회귀**: #1·#2 기존 테스트 그린(list_claims 필터 추가에도 기존 케이스 유지).
