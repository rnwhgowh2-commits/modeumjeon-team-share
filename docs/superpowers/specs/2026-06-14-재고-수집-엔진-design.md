## 1. 목적
무신사 르무통 메이트 등 상품의 **재고(품절 / 한정수량 / 표시없음)** 를 "현재 열린 사용자 브라우저 기준으로만" 수집·가공해 **사이즈별로** 정확히 도출 → 6소싱처 일반화.

## 2. 대상·범위
1순위: 무신사(source_id=3), 르무통 메이트. 테스트 상품 = `musinsa.com/products/4046672`(모음전·다색), `4800825`(크림핑크 전용), `6020529` 등. 1개 완성 후 6소싱처 일반화. 본 작업 = **재고편**(가격편 후속).

## 3. 핵심 원칙 (절대 규칙)
- **3-1. 현재 브라우저 기준만** — 로그인 상태마다 재고 표기가 달라지므로 지금 열린 브라우저 크롤값으로만.
- **3-2. 자동화 우선** — 확장이 사이즈별 재고를 자동 수집→저장. 수동 조회/붙여넣기 ❌. 조사도구 = 생산수집기 동일물.
- **3-3. 폴백·둔갑 금지** — 재고를 모르면 "충분(999)"으로 단정 ❌ → "재고불명/미수집"으로 표면화. (품절을 "있음"으로 팔면 오발주 = 금전 손실)

## 4. 문제 3가지 (현상 → 코드 원인)
- **4-1 한정수량 둔갑**: 240=2개인데 화면 "재고있음". 원인 = ext_bridge `_crawlItemInTab` 이 사이즈별 `options[]` 를 버리고 **상품레벨 stock(999)만 전송** → 전 사이즈 충분. (비무신사는 parse 결과를 **재고 합계**로 뭉개 24935 등.)
- **4-2 인벤토리 실패 → 999 둔갑 (가장 심각)**: 재고 API 호출 실패 시 확장이 **999(충분) 기본값** → 품절을 "있음"으로 둔갑(조용한 실패). 표면가엔 신선도 게이트 있으나 재고엔 없었음.
- **4-3 중복 리스팅 stale 마스킹**: 같은 상품 2 listing(4046672 모음전 / 4800825 전용) 중 한쪽 stale 999가 다른 쪽 품절을 가림(`pickBest` 가 in-stock 우선).

## 5. 재고 로직
- **3단계**: 품절(0) / 한정수량(실수량 N) / 표시없음(충분 = 999·null).
- **사이트별 마커 상이**:
  - 무신사: DOM `잔여 N개`·`N개 남음`·`마지막 N개` / API `remainQuantity`·`outOfStock`
  - SSF: `품절임박 (N)` / `statcd=SLDOUT`
  - SSG: `usablInvQty`(가용재고 정수)
  - 르무통·스스: `stock_number`(실재고) / `is_selling=F`
  - 롯데온·롯데아이몰: `disabled`/`li.soldout` — **수량 미제공**(있음/품절 2단계만)
- **stock_rules**: `soldout_markers[]` / `qty_patterns[]` / `no_marker_means(in_stock|unknown)` — 소싱처 정답카드(crawl_guide)에 명시.

## 6. 확정 결정
- **6-1** 재고 = 크롤 우선(remainQuantity·실수량 직읽기). 999·null = 충분 센티넬.
- **6-2** 인벤토리/파싱 실패 → **null(불명)** 전송, 서버가 **옛 좋은 값 보존(덮어쓰기 금지)**.
- **6-3** 수집방식 = API(prioritized-inventories) + DOM 마커. **단일색 상품은 사이즈로 매칭**, 다색은 색+사이즈.
- **6-4** 신선도 = TTL 없음, 마지막 크롤값 유지 + 크롤 시각 표시. **재크롤 시 stale 사이즈 prune**.
- **6-5** 무신사 먼저, 자동화 우선이라 확장 수집기 먼저.

## 7. 해결 아키텍처
- **7-1 통찰**: 사이즈별 재고는 이미 확장이 정확히 계산(remainQuantity)했음 — **전송·저장만 끊겨** 둔갑. 배선만 잇는 작업.
- **7-2 흐름**: 확장(사이즈별 stock — API/DOM 마커, 실패 시 null) → `crawl-result` `options[{color,size,stock}]` → `_ingest_option_stocks`(SourceOption.current_stock 색·사이즈 매칭) + `_prune_stale_option_sizes` → `_resolve_stock`(품절/N개/있음) → 매트릭스(소량 ≤4 주황).
- **7-3 조인키 계약**: 옵션 `(color, size)` ↔ SourceOption `(color_text, size_text)`, 공백·단일색 정규화(`_stk_cnorm`/`_stk_digits`).
- **7-4 저장**: 새 컬럼 ❌(기존 `SourceOption.current_stock`), prune = `deleted_at`(soft-delete).
