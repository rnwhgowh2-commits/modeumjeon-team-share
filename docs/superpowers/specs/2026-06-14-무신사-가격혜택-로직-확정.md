# 무신사 가격·혜택 로직 — 확정 명세 (2026-06-14, 사용자 정의)

> 단일 진실 원천: 무신사(source_id=3) 표면노출가·혜택·최종매입가 계산.
> 역할: **수집(확장) = URL의 정확한 정보를 프로그램에 넣기 / 로직(계산) = 프로그램(compute_final_price)**.
> 연관: `...-design.md`(엔진 설계), `_phase0/musinsa_benefit_keywords.md`(데이터).

---

## 0. 대원칙 (사용자 강조)

- **현재 열린 브라우저 URL 기준으로만** 표면노출가−혜택 = 최종매입가 결정.
- **데이터 신선도**: 표면가·혜택 모두 **현재 크롤값**. 크롤 없으면 미수집(폴백·템플릿·옛값 금지).
- 계산 로직은 **프로그램이 보유**(베이스금액 누적 등) — 사람/수집기가 재정의하지 않음. 수집기는 정보만 정확히 넣는다.

## 1. 표면노출가 (base)

- **표면노출가 = `goodsPrice.salePrice`** (무신사 API).
  - = 정가(normalPrice) × (1 − 선반영할인율 `discountRate`). 예: 179,000 × (1−9%) = 162,000.
  - ⚠️ **쿠폰 자동적용 상품은 UI에 salePrice가 안 보임**(정가→나의할인가로 점프). API/임베디드 데이터에만 존재 → **반드시 API에서 읽는다**(화면 회원가 '나의 할인가'는 base 아님).
  - 검증 게이트: salePrice>0 AND salePrice≤normalPrice. 실패 시 price=null(크롤실패, 폴백 금지).

## 2. 혜택 (표면가에서 베이스금액 순차 누적 차감 — 프로그램 compute_final_price)

| 혜택 | 금액 출처 | 비고 |
|---|---|---|
| 등급 할인 | 크롤값(트리거 '등급 할인' 뒤 금액) | 화면 '등급 할인 불가' → 미적용(0) |
| 상품 쿠폰 | 크롤값(트리거 '상품 쿠폰' 뒤 금액) | '쿠폰 없음' → 미적용(0) |
| 구매 적립 | 크롤값(트리거 '구매 적립' 뒤 금액) | **선할인 적용돼도 구매적립으로 가정** |
| 후기 적립 | **고정 500원** (가이드 value=500) | 텍스트후기만. 사진후기 2,500은 제외(크롤값 안 씀) |
| 결제수단 적립 | 크롤값(트리거 **'무신사머니 결제'** 뒤 금액) | "무신사머니 결제 시 X% 적립"만. ↓ 없으면 현대카드 |
| (대체) 현대카드 2.73% | 정률 0.0273 | **무신사머니 결제적립이 없을 때만** 적용. 직전잔액(≈최종매입가 직전)에 적용. 택1 |

- **계산식**: 최종매입가 = 표면가 − Σ(적용 혜택). 정액은 단순 차감과 동일, **현대카드(정률)는 직전잔액에 적용**(베이스금액 누적 — 프로그램이 처리).
- **수집 ≠ 계산 구분(중요)**: "무신사 **삼성카드** 결제 시 무신사머니 포인트 10% 적립" 같은 카드결제 라인은 **수집만 하고 계산엔 미사용**. 계산은 "**무신사머니 결제 시**"만.

## 3. 부재(不在) 인지 — "없으면 없다"

- 각 혜택의 상태(값/없음)를 빠짐없이 수집: 값(원/%) OR 부재신호(없음/불가/적용안함/품절/사용불가/미적용).
- 게이트(`gate_benefits`): 트리거 라인에 부재신호(exclude) 있으면 off, 트리거 라인 자체가 없어도 off. → 없는 혜택은 차감 0.

## 4. 신선도 (현재 브라우저 기준 보장)

- 매 크롤이 현재 페이지(로그인 상태 그대로)의 표면가+혜택 라인을 수집 → `dynamic_benefits_json['_crawl']` 스냅샷(crawled_at·is_logged_in·lines).
- `benefits_fresh`: 스냅샷 있고 benefits_ok=True → 그 스냅샷으로 계산. 없으면 **미수집**(final_price=null), 폴백 금지.
- 크롤 실패(error) 시 스냅샷 미갱신 → 마지막 성공 크롤값 유지 + 화면에 크롤 시각 표시(N1). 옛 템플릿/타-소싱처 값 사용 금지.
- 같은 SKU가 여러 무신사 SP에 매핑되면 `_crawl` 보유 SP 우선 선택.
