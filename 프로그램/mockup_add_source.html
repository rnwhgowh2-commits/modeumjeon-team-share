💡 시안 E 동작 원리

  1. 서버가 사용자 brower 대신 URL fetch (Playwright)
  2. HTML + 스크린샷 동시 추출
  3. Claude/GPT 가 셀렉터 추출 (vision + text)
  4. 동일 도메인 다른 URL 2개에 셀렉터 적용해 일관성 검증

시안 특징 한줄 ⏱ 입력 🛠 처리 🎯 정확도 📈 자동 수집 💼 적합 상황
A — 메타 등록 도메인·이름·로고만, 가장 단순 1~2분 즉시 — ❌ (URL 만) v1 출시 / URL 만 기록
B — 위저드 5단계 사용자가 셀렉터 직접 매핑 10~15분 즉시 ⭐⭐⭐ ✅ (셀프) 기술 사용자 / 완전 자율
C — 스크린샷 + AI ★ 화면 1장 → AI 자동 추출 3~5분 3초 (AI) ⭐⭐⭐⭐ ✅ (AI) 비기술 사용자 / 일반적
D — 개발자 티켓 요청 → 24h 내 완성 2분 24h ⭐⭐⭐⭐⭐ ✅ (개발자) 중요 소싱처 / 정확도 최우선
E — URL 자동 ★ URL 1개 → 모든 게 자동 30초 1~2분 ⭐⭐⭐⭐ ✅ (자동) 가장 빠름 / 일반적

📋 추천 조합 (사용자 워크플로 별)

  • 최선 (UX↑ + 자동화↑): 시안 E (URL 자동) — 사용자 부담 ↓, 정확도 ↑, ETA 1~2분
  • 차선 (실패 시 fallback): 시안 E → 실패 시 시안 D (개발자 티켓) 자동 큐잉
  • v1 빠른 출시: 시안 A (메타만) → 추후 시안 C / E 추가
  • 전문가용: 시안 B (위저드) — 사용자가 모든 셀렉터 직접 관리
