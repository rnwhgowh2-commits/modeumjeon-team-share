# 🗺️ 크롤링 가이드 — 데이터·코드 지도 (정본)

> **이 문서의 정체**: 모음전이 소싱처에서 **재고·가격을 어떻게 긁고, 어디에 저장하고, 어떻게 계산해, 어디에 표시**하는지를 한 곳에 적은 단일 진실 원천(SSOT).
>
> **읽는 법 (2명의 독자)**
> - 🟢 **사용자** — 각 절의 `🟢 쉬운 설명`만 읽어도 전체 그림이 잡힙니다.
> - 🔧💻 **Claude Code** — 방향을 잃으면 **이 문서를 위에서 아래로 그대로 읽고** 복귀하세요. `🔧 상세 로직` + `💻 핵심 코드(파일:라인)`까지 읽으면 코드 위치가 바로 잡힙니다.
>
> **갱신 규칙**: 코드가 바뀌면 이 문서의 해당 `💻` 블록도 같이 고칩니다. 파일:라인은 표류할 수 있으니 인용 시 실제 코드와 대조하세요. (작성 기준 2026-06-22)

## §0. 전체 흐름 지도

🟢 **쉬운 설명** — 소싱처 상품 하나의 재고·가격이 화면에 뜨기까지 **4단계**를 거칩니다.

```
[소싱처 사이트]
     │  ① 크롤러가 페이지에서 값을 긁음 (재고·표면노출가·혜택)
     ▼
[② 저장]  SourceProduct / SourceOption 테이블 + dynamic_benefits_json
     │
     ▼
[③ 계산]  표면노출가 − 혜택 = 최종매입가
     │
     ▼
[④ 표시]  매트릭스 셀 · 계산식(영수증) 팝업 · 크롤 로그
```

🔧 **상세 로직 + 💻 4개 층의 파일 위치**

| 층 | 하는 일 | 핵심 파일 |
|---|---|---|
| ① 크롤러(파싱) | 페이지 → 재고·가격·혜택 값 추출 | `lemouton/sourcing/crawlers/<소싱처>.py` |
| ② 저장(영속) | 옵션 dict → DB 컬럼 + `dynamic_benefits_json` | `lemouton/sources/service.py` (`PRODUCT_DYNAMIC_KEYS` :429 / `DYNAMIC_KEYS` :519) |
| ③ 계산(매입가) | 표면가 − 혜택 누적 차감 | `webapp/routes/api_benefits.py:404` `compute_breakdown` + `lemouton/pricing/unified.py:75` `compute_market_price` |
| ④ 표시(매트릭스) | 셀·로그·영수증 렌더 | `webapp/routes/api_pricing.py:309` `_option_matrix_data` + `webapp/templates/bundles/_matrix_v3.html` |

> ⚠️ **라이브 크롤은 크롬 확장(navGrab) 경로** — `extension/moum-crawler/background.js` → 서버 `/api/sources/parse`(파싱) → `/api/sources/crawl-result`(저장). 이 라이브 경로가 ②의 일부(혜택 저장)를 우회하는 문제가 있음 → §5 참조.

## §1. 재고편 — 소싱처별 재고 수집

🟢 **쉬운 설명** — 소싱처마다 재고를 알아내는 방법이 다릅니다. 어떤 곳은 페이지 코드 안에 "남은 수량"이 숫자로 들어있고(SSG), 어떤 곳은 "품절" 표시만 있고(롯데온), 어떤 곳은 별도 API를 부릅니다(무신사). 공통으로 **0=품절 / 실수량 / 999=수량 미상("재고있음")** 으로 정규화하고, 화면엔 **충분 · 소량(3개↓) · 품절 · 크롤실패 · 미크롤** 로 표기합니다.

🔧 **상세 — 재고 센티넬(약속된 숫자)**
- `0` = 품절 / `999` = 수량 미상 / 무신사 `STOCK_CAP` = 충분
- 종착지 `_resolve_stock`에서 `stock_label / stock_qty / stock_out`로 해석 — 💻 `webapp/routes/api_pricing.py:142`

### 소싱처별 재고 비교표

| 소싱처 | 수집 방식 | 재고 판정 규칙 (실코드) | 데이터 위치 | 파일 |
|---|---|---|---|---|
| **무신사** | API(POST) | `MusinsaCrawler.fetch` → `POST goods-detail.musinsa.com/api2/goods/{goodsNo}/options/v2/prioritized-inventories` · `outOfStock`→0 / "N개 남음"→실수량 / 없음→999 | `last_stock` | `crawlers/musinsa.py:492` |
| **SSG** | SSR `<script>` JS | `SsgCrawler.crawl` → `_parse_uitem_options` · 정규식 `usablInvQty`: `'0'`→품절, else 실수량 | `last_stock` | `crawlers/ssg.py:608` |
| **SSF** | HTML(curl_cffi) | `SsfCrawler.fetch` · `#optionDiv1 li a[optcd]@statcd`: `SLDOUT`→품절 / `품절임박 (N)`→실수량 | `current_stock` | `crawlers/ssf.py` |
| **롯데온** | API 우선→DOM 폴백 | API `pbf.lotteon.com/.../option/mapping/{spd}/{sitm}` 우선=실수량 → DOM `div.layer_option li.soldout`→0·else 999 | `last_stock` | `crawlers/lotteon.py:482,1293` |
| **스마트스토어** | 확장 per-SKU(로그인) | `/n/v2/channels/{cu}/products/{channelProductNo}`(=`A.id`) → `sku_stock`(색·사이즈별 0=품절·N=실수량) → `_persist_option_stocks` 영속 | `current_stock` | `ss_lemouton.py` + `api_pricing.py` |
| **르무통 공홈** | HTML(Cafe24) | `LemoutonCrawler` · `option_stock_data` 파싱 → 실조합·실재고만 | `current_stock` | `crawlers/lemouton.py` |

⚠️ **재고편 함정**
- 무신사 "마지막 N개" 정규식 빠지면 한정재고가 '재고있음'으로 둔갑.
- SSF 정규식 괄호 공백 `품절임박 ( N )` 처리 필수.
- 네이버(스마트스토어)는 `A.id`(channelProductNo) 사용(productNo 쓰면 204→999 둔갑).
- **확장이 per-SKU 재고를 잘 긁어도 `current_stock`에 저장 안 하면 매트릭스가 전부 999 둔갑.** 옵션단위 `current_stock`은 ① 서버사이드 `_ingest`(무로그인) ② 확장 `crawl-result`의 `_persist_option_stocks`(로그인 전용) **둘 다**에서 채워져야 함. 확장 전용(스스·SSF·SSG)은 ②가 단일 경로. (`[[project_smartstore_stock_not_persisted_extension_path]]`)
- 크롤 실패 시 **옛 재고 절대 금지**, "크롤 실패" 표기 → §4.

## §2. 가격편 — 소싱처별 가격·혜택 수집

🟢 **쉬운 설명** — 모든 소싱처의 매입가는 한 공식입니다.

> **표면 노출가 − 혜택(적립·할인·쿠폰·카드) = 최종 매입가**

%혜택은 **항상 베이스금액 기준**(직전 잔액에 곱함). 혜택 4분류: **선반영(preapplied)** · **후반영(deduct/accrue)** · **결제(payment, 택1)** · **캐시백(cashback)**.

💻 **계산 엔진**: `api_benefits.py:404 compute_breakdown` → `final_price.py compute_final_price` → `unified.py:75 compute_market_price`

### 소싱처별 계산 로직 (베이스금액① = 그 소싱처가 노출하는 기준가)

- **SSG** (`api_benefits.py:570~674·764`): 최적가(bestAmt, 즉시할인 선반영) → −SSG MONEY 적립(①무조건=항상 / ②충전결제=rate≥3%만) → −카드혜택가 → −상품쿠폰 / 현대카드 2.73% 청구할인 fallback(네이버페이 제외)
- **무신사** (`:727~752`): 회원가(surface_price) → −등급할인 → −등급적립 → −무신사머니 결제적립 → −상품쿠폰 + 후기적립(500원 고정) / 결제 택1: 무신사머니 3% vs 현대카드 2.73%
- **롯데온** (`:703~726·764`): 나의 혜택가(즉시·카드할인 선반영) → −롯데오너스 회원할인 → −스토어찜 쿠폰(토글) → −L.POINT 적립 → −OK캐시백 1.1% / 결제 택1: L.PAY·네이버페이 + 현대카드 fallback
- **SSF** (`:554~568`): 판매가(시즌 할인 선반영) → −기프트포인트 10%(멤버십, 즉시할인) → −멤버십포인트 적립(0.5~5%) / 결제 택1: 토스페이 5%
- **롯데아이몰** (`:675~702`): 최대할인가(할인+카드 청구할인 선반영) → −L.POINT 적립 / 무이자 할부
- **스마트스토어·르무통 공홈**: 표면 노출가 → 별도 동적혜택 적음 → 소싱처 템플릿값 사용

💻 **핵심 — SSG MONEY 적립 2분기** (`api_benefits.py:581`)
```python
_is_charge = ('충전' in _smt)                       # ② 충전결제 전용 판정
_ssgm_enabled = (not _is_charge) or (_rate >= 0.03)  # ①항상 / ②3%↑만
```
저장 키 화이트리스트: `service.py:432` (`ssg_money_rate`·`ssg_money_already_applied`·`ssg_money_text` 등).

### 💾 저장 배선 — 혜택이 어디서 `dynamic_benefits_json`에 박히나 (복구 핵심)

| 경로 | 저장 함수 | 혜택(`dynamic_benefits_json`) |
|---|---|---|
| **Python 서버 크롤** (fetch_one_source/배치) | `upsert_source_product` · `PRODUCT_DYNAMIC_KEYS:429` / `DYNAMIC_KEYS:519` | ✅ **저장됨** |
| **라이브 확장 크롤** (BG_PARSE→서버) | `save_crawl_result` (api_pricing.py:1011) | ❌ **미저장** (price·stock만) |

→ SSG/SSF 혜택(SSG MONEY 적립 등)은 라이브 확장 크롤로 **영원히 갱신 안 됨**. 혜택이 계산식에 안 뜨면 여기부터.

⚠️ 매칭/크롤 실패 시 **대표가(평균·최저) 폴백 절대 금지**. 라이브 확장 크롤은 SSG/SSF 등 혜택 키를 저장하지 않음(price/stock만) → §5.

## §3. 매트릭스 보기 > 로그의 가격·재고 표기

🟢 **쉬운 설명** — 모음전 상세 > 옵션 셀에서 가격·재고를 보는 곳은 **3군데**(셀 · 매트릭스 보기 팝업 · 크롤 로그)이고, 값은 **두 종류**입니다.
- **표면 노출가** — 소싱처에 적힌 그대로(셀 큰 숫자, 크롤 로그 "표면"=surf 취소선)
- **최종 매입가** — 혜택 다 뺀 실제 사는 값(계산식 영수증·셀 fx뱃지, 크롤 로그 "매입"=buy)

🔧💻 **데이터 위치·공유·fx 계산**

| 표기 | 무엇(필드) | 데이터 출처 | 파일 |
|---|---|---|---|
| 표면 노출가(셀) | `crawled_price` | `SourceProduct.last_price` / `SourceOption.current_price` | `api_pricing.py:395` |
| 최종 매입가(영수증·fx뱃지) | `final_price` | `compute_breakdown` / `compute_market_price` | `api_benefits.py:404` · `/api/source-benefits/breakdown` |
| 재고(셀) | `stock_label·stock_qty·stock_out` | `_resolve_stock`(센티넬 해석) | `api_pricing.py:142` |
| 크롤 로그 surf/buy | 표면=surf, 매입=buy | 확장 → CustomEvent | `ext_bridge.js` → `crawl_log.js` |

- **공유**: 셀 + 매트릭스 보기는 **같은 `window.DATA`** 한 벌을 읽음(로드 시 1회 `/api/bundles/<code>/option-matrix` fetch 캐시). 셀 최종가 = `window.SM_BREAKDOWNS["{sku}|{srcId}"].final_price`. **크롤 로그만 별도 실시간 스트림**(background.js surf=out.price → ext_bridge.js → crawl_log.js).

### 🧩 셀 · 매트릭스 보기 · 로그 — 위치 / 형태 / 호출처

| 요소 | ① 위치 (파일·DOM) | ② 형태 | ③ 호출처 |
|---|---|---|---|
| **📍 셀** | `_matrix_v3.html ~5734` · 옵션 탭 매트릭스의 소싱처 컬럼 | 소싱처별 표면노출가 + 재고 + **fx뱃지**(최종매입가) + ★최저 / 왼쪽 소싱참고·사입 박스 | `loadMatrix` → `/option-matrix` 1회 fetch → **window.DATA** 렌더 · fx=`SM_BREAKDOWNS["{sku}|{srcId}"]` |
| **🔲 매트릭스 보기**(팝업) | `_matrix_v3.html render() 1269~1374` · '매트릭스 보기' 버튼 | 행=옵션 × 열=소싱처 격자 + **상태점**(충분·소량3개↓·품절·크롤실패·미크롤) | 버튼 → render()가 **같은 window.DATA**(셀과 공유, 추가 호출 없음) |

## §4. 무결성 원칙 (방향 복귀 기준)

데이터 정확성 = 금전 손실 직결. 방향 잃으면 이 4개로 복귀.
1. **폴백가 금지** — 실패 시 평균·최저 대표가로 채우지 않는다. "가격없음 + 크롤실패"로 표면화.
2. **크롤 실패 = 표기** — 옛 가격/재고 끌어쓰지 않고 "크롤 실패". (하드리셋: 시작 NULL·pending → finalize)
3. **단일 진실 원천** — 표면가=`last_price`, 매입가=`compute_market_price` 한 곳에서만.
4. **누락에 경고** — 후보가 조용히 탈락(None)하면 안 됨. 화면·로그에 드러나야 함.

## §5. 자주 재발하는 문제 + 복귀 체크리스트

| 증상 | 첫 확인 위치 |
|---|---|
| 혜택(적립/할인)이 계산식에 안 뜸 | `dynamic_benefits_json`에 키 있나 → 없으면 라이브크롤 미배선 |
| 가짜 '재고있음'(품절인데 있음) | 999 둔갑? 정규식 미스? `_resolve_stock` |
| 표면가 ≠ 매입가 혼동 | §3 두 갈래 진실 (셀=표면 / 영수증=매입) |
| 크롤 후 매트릭스 갱신 안 됨 | `window.loadMatrix` 노출 / `SM_BREAKDOWNS` 캐시 |
| dev에서 값 이상 | dev DB는 stale일 수 있음 → 라이브 대조 필수 |

**알려진 구조적 결함**: SSG/SSF 혜택이 라이브 확장크롤로 갱신 안 됨 — `toItemBG`(background.js:1025)·`save_crawl_result`(api_pricing.py:1011)가 **price·stock만 저장**하고 `dynamic_benefits_json`(ssg_money 등) 미저장. + 서버 단일 재크롤 `refetch`도 `_detect_site_from_url`이 ssg.com→'ssf' 오분류(api_pricing.py:1481)·크롤러dict에 'ssg' 키 없음(1658).
