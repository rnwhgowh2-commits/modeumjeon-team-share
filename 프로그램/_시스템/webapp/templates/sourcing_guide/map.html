핵심재고·가격이 화면에 뜨기까지 4단계 — 크롤 → 저장 → 계산 → 표시.
01크롤러
페이지에서 재고·표면노출가·혜택을 추출한다.
코드crawlers/<소싱처>.py 뜻크롬 확장이 실행해 긁고 서버로 전송(추출만).
02저장
추출값을 DB 컬럼 + dynamic_benefits_json에 기록한다.
코드service.py:429 위치last_price·current_stock·dynamic_benefits_json (Supabase/AWS) 뜻긁은 값을 DB에 영속.
03계산
표면노출가 − 혜택 누적차감 = 최종매입가.
코드api_benefits.py:404 (compute_market_price) 뜻표면가에서 혜택을 순서대로 빼는 계산 엔진.
04표시
셀 · 매트릭스 보기 · 크롤 로그에 보여준다.
코드api_pricing.py:309 뜻매트릭스 렌더. 셀·매트릭스가 window.DATA 한 벌 공유.
⚠️ BG_PARSE(르무통·SSF·SSG·스스)는 price·stock만 저장 — 혜택 미저장.
📖 용어 사전 — 데이터를 읽는 3가지 방법
API 호출
사이트가 데이터를 나르는 내부 요청을 그대로 부름. 값(JSON)을 통째로 정확히 받음. 가장 빠르고 안정적 → 선호.
HTML 파싱
서버가 처음 보내준 화면 원본 문서(HTML)에서 값을 찾아 뽑음. 화면 구조가 바뀌면 깨질 수 있음.
DOM 읽기
브라우저가 HTML을 그려 만든 살아있는 화면(DOM)에서 읽음. JS로 나중에 채워지는 값까지 잡음. (HTML의 브라우저판)
핵심소싱처마다 재고·가격을 읽는 방법이 다르다 — API일수록 정확. 종착지 _resolve_stock.

🟢 같은 소싱처(예: SSG)에 URL이 2개 이상이면 셀은 그중 최종매입가가 가장 낮은 URL(후보)을 골라 표시. 표면가가 같아도 한 URL만 SSG MONEY 같은 혜택이 잡혀 더 쌀 수 있음(예 110,796).
철칙
셀과 계산식은 반드시 같은 후보를 써야 함. 셀은 bulk /breakdowns에 후보별 source_product_id 전달(최저 선택). 계산식(fx 팝업)은 단건 /breakdown/{sku}/{src}에 &source_product_id=<셀 best spid> 첨부 필수.
안 하면
셀(110,796) ≠ 계산식(116,627) — 백엔드가 등록 URL로 폴백. §4 단일 진실 원천 위반. 코드: 셀 data-cell-spid(5691) → fx ~3393·4017 → 백엔드 compute_breakdown 경로3(:523)
유령가 가드
최저 후보가 stale면 실제보다 싼 값 표시 → 폴백가 금지. best 후보는 last_status≠error·재고있음만. 의심 시 재크롤.
