📖 용어 사전 — 데이터를 읽는 3가지 방법
API 호출
사이트가 데이터를 나르는 내부 요청을 그대로 부름. 값(JSON)을 통째로 정확히 받음. 가장 빠르고 안정적 → 선호.
HTML 파싱
서버가 처음 보내준 화면 원본 문서(HTML)에서 값을 찾아 뽑음. 화면 구조가 바뀌면 깨질 수 있음.
DOM 읽기
브라우저가 HTML을 그려 만든 살아있는 화면(DOM)에서 읽음. JS로 나중에 채워지는 값까지 잡음. (HTML의 브라우저판)
핵심소싱처마다 재고·가격을 읽는 방법이 다르다 — API일수록 정확. 종착지 _resolve_stock.
왜 소싱처마다 다른가 — 4가지가 방식을 정함: ① 전용 재고 API를 공개하나(무신사·롯데온) ② 페이지에 내장 구조데이터(__NEXT_DATA__·option_stock_data·uitemObj)가 있나(현대H몰·르무통·SSG) ③ WAF/봇차단으로 서버 접근이 막히나(롯데아이몰·네이버) ④ 로그인은 대개 혜택만 필요(재고·표면가는 무로그인). 그래서 "API > 내장 JSON > HTML > DOM" 순으로 가능한 가장 높은 방식을 쓴다.
