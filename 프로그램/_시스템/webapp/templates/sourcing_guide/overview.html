📖 신규 소싱처 크롤링 — 상세 로직 (Claude 실행 SOP)

STEP 2 · 수집 방식 4분면 분류 (어떻게 긁나 × 로그인 필요한가)

비인증(공개)인증 필요(로그인)
API 방식가장 쉬움 — 로그인 없이 내부 API(JSON) 호출로그인 세션 + 내부 API (예: 무신사 = 회원가·혜택형)
HTML 방식API 없음 — 공개 화면(DOM) 파싱가장 까다로움 — 로그인 + 화면 파싱
→ API 크롤링(내부 요청 직접 호출)이 빠르고 안정적이라 우선. HTML 스크래핑은 화면 바뀌면 깨짐.

STEP 3 · 항목별 수집 양식 — 6항목을 2축(수집 방식 × 인증)으로 분류 (예: 무신사)

⚠️ 옵션·재고는 위치만 적고 끝내지 않는다 — 재고 3단계를 명시. ① 품절(마커 → 0) · ② 한정수량(잔여·N개 남음·마지막·품절임박 등 표기 전부 → 실수량 N) · ③ 표식없음(표식 정말 없을 때만 '재고있음'). ②를 한 가지만 적고 다른 표기를 빠뜨리면 = 한정재고가 "재고있음"으로 둔갑하는 버그(실제 무신사 사례).
