?데이터 원본 차이 — HTML · API · DOM▾
원본 HTML
서버가 처음 보내주는 페이지 코드 자체(소스보기에 보이는 그것)에서 값을 읽는 방식
수집 특징추가 요청 없이, 코드·내장 JS변수에 박힌 값을 그대로 파싱. 로그인 불필요한 경우가 많음
왜 이렇게?재고·가격이 처음 HTML에 다 담겨 올 때(서버가 미리 그려 보냄). 가장 단순하고 빠름
API (JSON)
페이지가 따로 서버에 요청해 받아오는 데이터(JSON)를, 그 요청을 직접 불러 읽는 방식
수집 특징정확한 숫자(실재고 등)를 구조화된 형태로 받음. 로그인·인증·WAF 통과가 필요할 수 있음
왜 이렇게?화면 숫자가 원래 이 API에서 온 것 → 원본을 직접 읽어 가장 정확(화면은 이걸 가공한 결과)
DOM (화면)
브라우저가 화면에 다 그려낸 뒤의 결과(보이는 요소)에서 텍스트를 읽는 방식
수집 특징사람이 보는 그대로. 단 클릭·스크롤로 렌더돼야 보이고, "10개"처럼 캡·가공된 표시일 수 있음
왜 이렇게?API를 못 찾거나 막힐 때, 화면에 보이는 걸 직접 긁는 최후 수단
💡 데이터 소스 우선순위 API > 원본 HTML/JS > DOM · 🛡 전체크롤=하드리셋 · 크롤 실패 시 "크롤 실패" 표기 · 옛 가격/재고 절대 금지(폴백가 금지).
