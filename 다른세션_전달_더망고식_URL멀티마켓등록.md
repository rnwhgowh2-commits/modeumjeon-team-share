# 다른 세션 전달 — 더망고식: 소싱 URL → 가공규칙 → 멀티마켓 동시 등록

> 사장님 요구(2026-07-21): "소싱처 임의 상품 URL 을 프로그램에 넣으면, 가공 규칙에 따라
> 내가 고른 마켓들에 자동 업로드. 더망고가 이걸 한다 — 우리도 구현하고 싶다."
> ESM 연동 세션에서 접수했으나 대량등록 워크스트림과 파일이 겹쳐 **그쪽 세션에서 진행**.

## 사장님 질문에 대한 답(전달할 것)
"마켓별로 필요한 정보가 충분해야 업로드 되는 것 아니냐?" — **맞다.** 마켓마다 필수값이
다르다(카테고리·고시정보·인증·출하지·배송정책·옵션 규격…). 더망고가 되는 이유는
① 크롤값(이름·가격·이미지·옵션)으로 채울 수 있는 건 채우고
② 나머지 필수값은 **계정·카테고리별 기본값 템플릿**으로 자동 보충하기 때문.
우리의 「가공 규칙 13항목」이 정확히 그 ②를 담당하도록 설계돼 있다.

## 이미 있는 부품 (재구현 금지)
- 대량등록 Phase 1A ✅라이브: 수기입력 → 스스·쿠팡 등록(`/bulk`, compile_common)
- 가공 규칙 13항목: 스키마+편집 화면 main 반영(2026-07-21, 125b78f0·0accb678)
- **4대 마켓 실등록 검증 완료**(2026-07-21 다른 세션): 옥션·G마켓·11번가·롯데온
  실제 등록→판매중지 사이클 성공. 등록 body 함정들 그 세션 메모리에 있음
- 소싱처 크롤: 크롬 확장(로컬) — URL 단위 크롤·옵션·가격·이미지 확보 경로 존재
- 마켓별 등록 API 지도: marketplace_api_map.json (ESM 함정 traps 포함)

## 빠져 있는 것(이번 작업 범위)
1. **URL 입력 → 크롤 → 등록 초안(draft) 매핑** 파이프라인
   (크롤 결과 → product_drafts 필드 채움. 크롤=로컬 원칙 준수 — 서버 크롤 금지)
2. **마켓별 필수값 검증기**: 등록 전에 "이 마켓은 X 가 비어서 못 올림"을 정직하게 표
   (조용한 실패 금지 — 더망고도 실패건은 사유를 보여준다)
3. **멀티마켓 동시 등록 오케스트레이션**: 마켓 선택 체크박스 → 계정별 순차/병렬 등록,
   건별 성공/실패 표(마켓 응답 원문 사유 포함)
4. 가공 규칙 적용 지점 연결(상품명 가공·가격 계산·금지어 등 13항목)
